Version Range: v4.5.6-beta.9 to v4.5.6-beta.14

Access Control:
- Introduced Permission Provider approach for centralized access control without syncing users or calling Permissions API directly
- Added `maxDepth` to [`Get Folders`](/api-reference/rest-apis/v2/folders/get-folders) REST API to retrieve nested subfolders at any depth
- Added `inheritFromParent` to [Update Folder Access](/api-reference/rest-apis/v2/folders/update-folder-access) to inherit parent permissions
- Added `accessType` to [Add Folder](/api-reference/rest-apis/v2/folders/add-folder) REST API to set permissions during folder creation
- [Update Folder](/api-reference/rest-apis/v2/folders/update-folder) now auto-updates `ancestors` and `accessType` when moving folders
- Resources inherit access type and permissions from parents; explicit settings override inherited values during evaluation and enforcement

Notifications:
- Added `triggerNotification` to [Add Comment Annotations](/api-reference/rest-apis/v2/comments-feature/comment-annotations/add-comment-annotations) REST API
- In-app notifications and APIs now only create/retrieve notifications for documents the user has access to
- Added `verifyUserPermissions` to [`Add Notifications`](/api-reference/rest-apis/v2/notifications/add-notifications) REST API
- Added `verifyUserPermissions` to [`Update Notifications`](/api-reference/rest-apis/v2/notifications/update-notifications) REST API

Core:
- `setDocuments` now filters inaccessible documents instead of failing the entire operation
- Folder document limit set to 50 with `allDocuments` flag; inaccessible documents are filtered out

Live State Sync:
- Fixed `getLiveStateData()` with `listenToNewChangesOnly: true` to return only new changes after subscribing
