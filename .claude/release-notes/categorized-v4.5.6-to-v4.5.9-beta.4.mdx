Version Range: v4.5.6 to v4.5.9-beta.4

Core:
- Added `getHeartbeat()` method to track user-specific heartbeat data for any user, providing visibility into active sessions
- Added `fetchDebugInfo()` and `getDebugInfo()` methods to retrieve real-time debugging info about your Velt implementation
- Optimized the initialization module and significantly reduced the SDK initialization time

Comments:
- Added preliminary infrastructure to capture screenshots in comments
- Enhanced `selectCommentByAnnotationId()` to close selected comment annotations when called with no arguments or invalid ID

Access Control:
- Launched Permission Provider to verify user access via your backend as source of truth [Learn more →](/key-concepts/overview#c-real-time-permission-provider)
- Added auto-revoke config to remove permissions when users log out or documents unset [Learn more →](/key-concepts/overview#c-real-time-permission-provider)
- Added Permission Provider events to monitor permission checks for debugging [Learn more →](/api-reference/sdk/models/data-models#permissionproviderevent)
- Added Access Context for granular feature-level permissions [Learn more →](/key-concepts/overview#set-feature-level-permissions-using-access-context-custom-metadata)
- Enforce Access Context restrictions preventing unauthorized users from adding comments, reactions, recorders, and area comments
- Simplified Permission Provider by removing client-side signature handling - now fully managed internally by SDK
- Added `source` field to Permission Provider requests to identify which method triggered the permission check

Recorder:
- Added video editor timeline image preview displaying frame snapshots for quick scene navigation
- Updated video editor timeline picture mode design for clearer visual distinction between selected and unselected frames
- Fixed audio merging to combine microphone and tab audio in screen recordings
