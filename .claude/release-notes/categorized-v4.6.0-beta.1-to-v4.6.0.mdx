Version Range: v4.6.0-beta.1 to v4.6.0

Core:
- Added `getCurrentUser()` method to subscribe to user object changes in real-time
- Added validation in `getUserPermissions()` to check if user is set before proceeding

Comments:
- Added `commentPinClicked` event to detect pin interactions and track user engagement
- Added `readOnly` property to Comments Sidebar for read-only mode to prevent edits
- Extended private comments feature to support both admin and organization users
- Added default access context to ensure relevant comments are fetched with feature-level permissions ([Learn More](https://docs.velt.dev/key-concepts/overview#set-feature-level-permissions-using-access-context-custom-metadata))

REST APIs:
- Added `verifyUserPermissions` flag to [Add Comment Annotations](/api-reference/rest-apis/v2/comments-feature/comment-annotations/add-comment-annotations), [Add Comments in Comment Annotation](/api-reference/rest-apis/v2/comments-feature/comments/add-comments), [Add Notifications](/api-reference/rest-apis/v2/notifications/add-notifications), and [Update Notifications](/api-reference/rest-apis/v2/notifications/update-notifications) REST APIs to prevent unauthorized resource creation or updates
