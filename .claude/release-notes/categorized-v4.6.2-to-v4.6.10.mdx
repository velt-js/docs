Version Range: v4.6.2 to v4.6.10

## Comments

### New Features
- Added mark as read/unread option in comment dialog to toggle read status for individual comments
- Added `unread` property to annotations to check if an annotation is unread by the current user
- Added Open and Reset filters to sidebar minimal filter dropdown to filter annotations by open status
- Added `context` prop to comment sidebar to pass context data to page mode composer comments
- Added `commentCountType` prop to control which count displays in sidebar buttons (total or unread)
- Added `openDialog` prop to comment bubbles to prevent dialog from opening when bubble is clicked
- Added `commentBubbleClickedEvent` to detect comment bubble interactions
- Added unread icon wireframes to sidebar buttons and comment bubbles for custom unread indicators

### Improvements
- Added paginated contact list to limit user downloads, reducing data transfer for apps with thousands of users
- Improved robust XPath detection logic when resolving elements through comment anchors for more scenarios
- Refactored Tiptap and Lexical Velt Comments libraries to follow common pattern for faster development
- Added `expandOnSelection` prop to Comments Sidebar to control whether dialogs expand when selected
- Added event subscription support for data providers via `Velt.on` method for monitoring and debugging
- Reply counts now display in read-only mode, showing number of replies on each comment
- Added `dialogSelection` prop to disable dialog selection in sidebar

### Bug Fixes
- Fixed sidebar button showing "..." instead of "0" when document has no comments
- Fixed `data-velt-comment-pin-active` attribute being incorrectly removed when dialog was still selected
- Fixed reset filter in multi-thread comments not showing resolved comments
- Fixed race condition in `addContext` where user context was being overwritten
- Fixed Comment Bubble closing unexpectedly after adding the first comment
- Fixed draft comment annotations not being created
- Fixed `contextId` generation in comment annotations for correct context filtering
- Fixed pin click event support for Standalone Comment Pin Component (VeltCommentPin)

## CRDT

### New Features
- Added webhook support for CRDT data changes with configurable debounce time (default 5 seconds)
- Added `updateData` event subscription to listen for CRDT data changes in real-time

## Access Control

### Improvements
- Added context information to permission provider requests when `setDocuments` is called
- Added context access information to `getUserPermissions` API response with `accessFields` array

## Core

### New Features
- Added `getCurrentUserPermissions` API to subscribe to current user permissions for debugging and developer tools

### Improvements
- Added batching for internal service calls, reducing network calls by 90%
- Added `throwError` config to `identify()` and auth provider methods for error handling control
- Improved `updateContext()` logic to handle context access scenarios correctly

### Bug Fixes
- Fixed document metadata not being saved during `setDocuments`

## Live Selection

### New Features
- Added `setInactivityTime()` to configure inactivity duration before removing stale live selections (default 5 min)

## UI Customization

### New Features
- Added unread icon wireframes to sidebar buttons and comment bubbles for custom unread indicators
