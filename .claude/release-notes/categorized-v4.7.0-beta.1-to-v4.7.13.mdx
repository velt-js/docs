Version Range: v4.7.0-beta.1 to v4.7.13

## Comments

### New Features
- Added `attachmentDownloadClicked` event and methods to intercept attachment download clicks for custom analytics, URLs, or access control
- Added `setContextInPageModeComposer()` and `focusPageModeComposer()` methods to set context data and focus the page mode composer programmatically
- Added `clearPageModeComposerContext()` method to clear context data from page mode composer when using sidebar in embed mode
- Added `commentToolClick` event that fires when the comment tool button is clicked
- Added `sidebarButtonClick` event that fires when the sidebar button is clicked
- Added ability to configure assign-to functionality using either a dropdown or checkbox mode for assigning users to comments
- Added `VeltCommentDialogThreadCardReactionPin` wireframe component to display specific reaction pins within comment thread cards
- Added `excludeReactionIds` on `VeltCommentDialogThreadCardReactions` and `VeltCommentDialogThreadCardReactionTool` to exclude specific reactions
- Added "Assigned to me" filter option to the comment sidebar minimal filter dropdown
- Added `batchedPerDocument` and `debounceMs` properties to `CommentRequestQuery` for batched per-document comment counts
- Added Thread card assign button wireframe to customize the assign button on individual comment thread cards
- Added `clearComposer()` method to reset composer state including text, attachments, recordings, tagged users, assignments, and custom lists
- Added `getComposerData()` method to fetch current composer state on-demand
- Added `submitComment()` method to programmatically trigger comment submission from custom buttons or keyboard shortcuts
- Added `composerTextChange` event to track text changes in comment composers for auto-save, character counters, or real-time validation
- Added `forceClose`, `sortOrder`, `sortBy`, and `defaultMinimalFilter` props to sidebar for controlling behavior and default filtering
- Added `context`, `locationId`, `documentId`, and `folderId` props to standalone Composer component
- Added `openAnnotationInFocusMode` config to open comment dialog in focus mode when `focusedThreadMode` is enabled

### Breaking Changes
- Renamed `targetElementId` prop to `targetComposerElementId` on `VeltCommentComposer`
- Changed `submitComment()` to accept a request object; migrate from `submitComment(targetElementId)` to `submitComment({ targetComposerElementId })`

### Improvements
- Added `readOnly` prop to `VeltCommentComposer` and `VeltInlineCommentsSection` to disable comment input at the component level
- Added edit composer wireframe for the comment dialog to customize the edit composer UI within comment thread cards
- Added ability to associate page mode composer comments with specific elements using the `targetElementId` property
- Added `CommentAnnotation.resolvedByUser` property exposing the full `User` object without requiring additional API calls
- Enhanced `composerTextChange` event to include full draft `annotation` object and `targetComposerElementId` for real-time composer state access
- Added `elementId` support in `/v2/commentannotations/add` REST API to define text search boundaries for text comments
- Context data now passed from sidebar and inline views to dialog, accessible via `velt-if` and `velt-data` directives


## Notifications

### New Features
- Added `pageSize` prop to control the initial notification load count in NotificationPanel and NotificationTool

### Improvements
- Added dropdown layout option for notification settings, allowing a choice between accordion or dropdown layouts


## Self-Hosting

### New Features
- Added config-based attachment resolver with `saveConfig` and `deleteConfig` endpoint URLs for uploads and deletions.
  [Learn more](/self-host-data/attachments)
- Introduced Config-Based DataProviders, reducing self-hosting implementation overhead by 90%
  - **Frontend**: Configure URLs and headers; the SDK handles requests automatically
  - **Backend**: Use the new [Velt Python SDK](https://pypi.org/project/velt-py) to pass DB and storage configs directly with no manual processing


## CRDT

### New Features
- Added [Get CRDT Data](/api-reference/rest-apis/v2/crdt/get-crdt-data) REST API to fetch document editors by ID or retrieve all editors.
  [Learn more](/realtime-collaboration/crdt/setup/core#backend-rest-apis)

### Improvements
- Added CRDT data in Velt Console Data tab for improved debugging and visibility into the CRDT store


## Access Control

### Improvements
- Added `context` field in document type during `setNotifications` call in PermissionProvider for additional context data when debugging


## Core

### Improvements
- Improved mutation observer element detection to skip animation-related elements, preventing performance slowdowns from continuous animations
- Previous document is now set on organization change when `identify()` is called, unless `setDocuments()` is explicitly called
