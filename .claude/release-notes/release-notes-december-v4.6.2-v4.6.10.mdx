Long time no chat Velt Developers!

Here's what shipped from `v4.6.2` until `v4.6.10` across Comments, CRDT, Access Control, Core, Live Selection, and UI Customization.

:speech_balloon: *Comments*

- Added mark as read/unread option in comment dialog to toggle read status for individual comments. [See wireframe customization](https://docs.velt.dev/ui-customization/features/async/comments/comment-dialog-components#optionscontentmarkasread).
- Added `unread` property to annotations to check if an annotation is unread by the current user. [See `CommentAnnotation` data model](https://docs.velt.dev/api-reference/sdk/models/data-models#commentannotation).
- Added Open and Reset filters to sidebar minimal filter dropdown to filter annotations by open status. [See wireframe customization](https://docs.velt.dev/ui-customization/features/async/comments/comment-sidebar-components#minimalfilterdropdown-panel-header).
- Added `context` prop to comment sidebar to pass context data to page mode composer comments. [Learn more →](https://docs.velt.dev/async-collaboration/comments-sidebar/customize-behavior#context)
- Added `commentCountType` prop to control which count displays in sidebar buttons (total or unread). [Learn more →](https://docs.velt.dev/async-collaboration/comments-sidebar/customize-behavior#commentcounttype)
- Added `openDialog` prop to comment bubbles to prevent dialog from opening when bubble is clicked. [Learn more →](https://docs.velt.dev/ui-customization/features/async/comments/comment-bubble#open-dialog-on-click)
- Added `commentBubbleClickedEvent` to detect comment bubble interactions. [Learn more →](https://docs.velt.dev/async-collaboration/comments/customize-behavior#commentbubbleclicked)
- Added paginated contact list to limit user downloads, reducing data transfer for apps with thousands of users. [Learn more →](https://docs.velt.dev/async-collaboration/comments/customize-behavior#enablepaginatedcontactlist)
- Added `expandOnSelection` prop to Comments Sidebar to control whether dialogs expand when selected. [Learn more →](https://docs.velt.dev/async-collaboration/comments-sidebar/customize-behavior#expandonselection)
- Added event subscription support for data providers via `Velt.on` method for monitoring and debugging. [Learn more →](https://docs.velt.dev/api-reference/sdk/api/api-methods#event-subscription-4)
- Reply counts now display in read-only mode, showing number of replies on each comment. [Learn more →](https://docs.velt.dev/async-collaboration/comments/customize-behavior#readonly)
- Added `dialogSelection` prop to disable dialog selection in sidebar. [Learn more →](https://docs.velt.dev/async-collaboration/comments-sidebar/customize-behavior#dialogselection)

:crystal_ball: *CRDT*

- Added webhook support for CRDT data changes with configurable debounce time. Default: 5 seconds. [See documentation](https://docs.velt.dev/realtime-collaboration/crdt/setup/core#webhooks).
- Added `updateData` event subscription to listen for CRDT data changes in real-time. [See documentation](https://docs.velt.dev/realtime-collaboration/crdt/setup/core#on).

:closed_lock_with_key: *Access Control*

- Added context information to permission provider requests when `setDocuments` is called. [Learn more →](https://docs.velt.dev/key-concepts/overview#filter-comments-by-context)
- Added context access information to `getUserPermissions` API response with `accessFields` array. [Learn more →](https://docs.velt.dev/key-concepts/overview#frontend-apis-7)

:gear: *Core*

- Added `getCurrentUserPermissions` API to subscribe to current user permissions for debugging and developer tools. [Learn more →](https://docs.velt.dev/api-reference/sdk/api/api-methods#getcurrentuserpermissions)
- Added batching for internal service calls, reducing network calls by 90%.
- Added `throwError` config to `identify()` and auth provider methods for error handling control. [Learn more →](https://docs.velt.dev/get-started/advanced#error-handling-in-authentication)
- Improved `updateContext()` logic to handle context access scenarios correctly.

:three_button_mouse: *Live Selection*

- Added `setInactivityTime()` to configure inactivity duration before removing stale live selections. Default: 5 minutes. [Learn more →](https://docs.velt.dev/realtime-collaboration/live-selection/customize-behavior#setinactivitytime)

:art: *UI Customization*

- Added unread icon wireframes to sidebar buttons and comment bubbles for custom unread indicators. [See sidebar button wireframes](https://docs.velt.dev/ui-customization/features/async/comments/comment-sidebar-button#unreadicon). [See comment bubble wireframes](https://docs.velt.dev/ui-customization/features/async/comments/comment-bubble#unreadicon).

For full API examples and details, check the [Velt SDK Changelog](https://docs.velt.dev/release-notes/version-4/sdk-changelog).

Happy shipping! :rocket:
