Hey Velt Developers! Hope you're enjoying March Madness!

Here's what shipped from `v4.7.0-beta.1` until `v4.7.13` across Comments, Notifications, Self-Hosting, CRDT, Access Control, and Core.

:speech_balloon: *Comments*

- Added `attachmentDownloadClicked` event and companion methods to intercept attachment download clicks for custom analytics, redirect URLs, or access control logic.
- Added `setContextInPageModeComposer()` and `focusPageModeComposer()` methods to set context data and programmatically focus the page mode composer.
- Added `clearPageModeComposerContext()` method to clear context data from the page mode composer when using the sidebar in embed mode.
- Added `commentToolClick` event that fires when the comment tool button is clicked.
- Added `sidebarButtonClick` event that fires when the sidebar button is clicked.
- Added ability to configure the assign-to UI using either a dropdown or checkbox mode for assigning users to comments.
- Added `VeltCommentDialogThreadCardReactionPin` wireframe component to display specific reaction pins within comment thread cards.
- Added `excludeReactionIds` on `VeltCommentDialogThreadCardReactions` and `VeltCommentDialogThreadCardReactionTool` to exclude specific reactions from rendering.
- Added "Assigned to me" filter option to the comment sidebar minimal filter dropdown.
- Added `batchedPerDocument` and `debounceMs` properties to `CommentRequestQuery` for batched per-document comment counts.
- Added thread card assign button wireframe to customize the assign button on individual comment thread cards.
- Added `clearComposer()` method to reset the full composer state, including text, attachments, recordings, tagged users, assignments, and custom lists.
- Added `getComposerData()` method to fetch the current composer state on-demand.
- Added `submitComment()` method to programmatically trigger comment submission from custom buttons or keyboard shortcuts.
- Added `composerTextChange` event to track text changes in comment composers, useful for auto-save, character counters, or real-time validation. Enhanced to include the full draft `annotation` object and `targetComposerElementId` for deep real-time composer state access.
- Added `forceClose`, `sortOrder`, `sortBy`, and `defaultMinimalFilter` props to the sidebar for controlling behavior and default filtering.
- Added `context`, `locationId`, `documentId`, and `folderId` props to the standalone `Composer` component.
- Added `openAnnotationInFocusMode` config to open the comment dialog in focus mode when `focusedThreadMode` is enabled.
- Added `readOnly` prop to `VeltCommentComposer` and `VeltInlineCommentsSection` to disable comment input at the component level.
- Added edit composer wireframe for the comment dialog to customize the edit composer UI within comment thread cards.
- Added ability to associate page mode composer comments with specific elements using the `targetElementId` property.
- Added `CommentAnnotation.resolvedByUser` property exposing the full `User` object without requiring additional API calls.
- Added `elementId` support in the `/v2/commentannotations/add` REST API to define text search boundaries for text comments.
- Context data is now passed from sidebar and inline views into the dialog, accessible via `velt-if` and `velt-data` directives.
- **Breaking change:** Renamed `targetElementId` prop to `targetComposerElementId` on `VeltCommentComposer`.
- **Breaking change:** `submitComment()` now accepts a request object — migrate from `submitComment(targetElementId)` to `submitComment({ targetComposerElementId })`.

:bell: *Notifications*

- Added `pageSize` prop to control the initial notification load count in `NotificationPanel` and `NotificationTool`.
- Added dropdown layout option for notification settings, allowing a choice between accordion or dropdown layouts.

:floppy_disk: *Self-Hosting*

- Added config-based attachment resolver with `saveConfig` and `deleteConfig` endpoint URLs for uploads and deletions. [Learn more](https://docs.velt.dev/self-host-data/attachments).
- Introduced Config-Based DataProviders, reducing self-hosting implementation overhead by 90%. Configure URLs and headers on the frontend — the SDK handles requests automatically. On the backend, use the new [Velt Python SDK](https://pypi.org/project/velt-py) to pass DB and storage configs directly with no manual processing required.

:crystal_ball: *CRDT*

- Added [Get CRDT Data](https://docs.velt.dev/api-reference/rest-apis/v2/crdt/get-crdt-data) REST API to fetch document editors by ID or retrieve all editors. [Learn more](https://docs.velt.dev/realtime-collaboration/crdt/setup/core#backend-rest-apis).
- Added CRDT data to the Velt Console Data tab for improved debugging and visibility into the CRDT store.

:closed_lock_with_key: *Access Control*

- Added `context` field to the document type during `setNotifications` calls in `PermissionProvider`, providing additional context data when debugging permission flows.

:gear: *Core*

- Improved mutation observer element detection to skip animation-related elements, preventing performance slowdowns caused by continuous animations.
- Previous document is now correctly set on organization change when `identify()` is called, unless `setDocuments()` is explicitly called.

For full API examples and details, check the [Velt SDK Changelog](https://docs.velt.dev/release-notes/version-4/sdk-changelog).

Happy shipping! :rocket:
