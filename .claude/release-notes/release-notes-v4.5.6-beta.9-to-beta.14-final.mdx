What's Up Velt Developers!

Here's what shipped from `v4.5.6-beta.9` until `v4.5.6-beta.14` across Access Control, Notifications, Core, and Live State Sync.

:closed_lock_with_key: *Access Control*

- Introduced a new [Permission Provider](https://docs.velt.dev/key-concepts/overview#c-real-time-permission-provider) approach for implementing centralized access control without syncing users in Velt or calling Permissions API directly
- Improved [access control model](https://docs.velt.dev/key-concepts/overview#access-control-model): Resources (Organizations, Folders, Documents) now follow a hierarchical permission model similar to Google Drive, where child resources inherit access settings from parents by default, but explicit settings on a resource override inherited values during evaluation and enforcement
- Added `maxDepth` to [Get Folders](https://docs.velt.dev/api-reference/rest-apis/v2/folders/get-folders) REST API to retrieve nested subfolders at any depth
- Added `inheritFromParent` to [Update Folder Access](https://docs.velt.dev/api-reference/rest-apis/v2/folders/update-folder-access) to inherit parent permissions
- Added `accessType` to [Add Folder](https://docs.velt.dev/api-reference/rest-apis/v2/folders/add-folder) REST API to set permissions during folder creation
- [Update Folder](https://docs.velt.dev/api-reference/rest-apis/v2/folders/update-folder) now auto-updates `ancestors` and `accessType` when moving folders

:bell: *Notifications*

- Added `triggerNotification` parameter to [Add Comment Annotations](https://docs.velt.dev/api-reference/rest-apis/v2/comments-feature/comment-annotations/add-comment-annotations) REST API to control whether notifications are sent when programmatically adding comments
- In-app notifications and APIs now only create/retrieve notifications for documents the user has access to
- Added `verifyUserPermissions` parameter to [Add Notifications](https://docs.velt.dev/api-reference/rest-apis/v2/notifications/add-notifications) and [Update Notifications](https://docs.velt.dev/api-reference/rest-apis/v2/notifications/update-notifications) REST APIs to validate user access before creating or updating notifications

:gear: *Core*

- `setDocuments` now filters inaccessible documents instead of failing the entire operation
- Folder document limit set to 50 with `allDocuments` flag; inaccessible documents are filtered out

:crystal_ball: *Live State Sync*

- Fixed `getLiveStateData()` with `listenToNewChangesOnly: true` to return only new changes after subscribing

:gear: *Velt Dev Tools*
- Added more improvements to DevTools to help you with debugging the Velt SDK locally: https://chromewebstore.google.com/detail/velt-devtools/nfldoicbagllmegffdapcnohakpamlnl

For full API examples and details, check the [Velt SDK Changelog](https://docs.velt.dev/release-notes/version-4/sdk-changelog).
