title: "Template Variables"
---

You can render dynamic data from these Data Classes inside your Velt Components Wireframe:

**1. [User](/api-reference/models/User.mdx):** This represents the current logged-in user. It is available across all components.

**2. [UserContact](/api-reference/models/User.mdx):** This represents the user contact object (it has the same class as the User). It is available on the autocomplete component where the user contacts are rendered.

**3. [CommentAnnotation](/api-reference/models/CommentAnnotation.mdx):** This represents the comment thread. This is available within the comment feature components.

**4. [Comment](/api-reference/models/Comment.mdx):** This represents the message inside the thread. This is available within the comment feature components.

Example Usage:

<Tabs>
<Tab title="React / Next.js">
```jsx
<VeltAutocompleteOptionWireframe>
  <VeltData path='userContact.name' />
  <VeltData path='userContact.organizationName' />
</VeltAutocompleteOptionWireframe>
```
</Tab>

<Tab title="HTML / Other Frameworks">
```jsx
<velt-autocomplete-option-wireframe>
  <velt-data path="userContact.name"></velt-data>
  <velt-data path="userContact.organizationName"></velt-data>
</velt-autocomplete-option-wireframe>
```
</Tab>
</Tabs>
