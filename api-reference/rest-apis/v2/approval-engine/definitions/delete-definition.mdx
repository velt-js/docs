---
title: "Delete Definition"
api: "POST https://api.velt.dev/v2/workflow/definitions/delete"
---

Use this API to delete a workflow definition. By default, the definition is **soft-deleted** (tombstoned): it is filtered from `get` and `list`, but its document and version history are retained. Pass `purge: true` to **hard-delete** the definition and physically remove its version history as well.

After either delete, the `definitionId` is immediately available for re-creation — recreating a definition over a tombstoned id succeeds. Definitions with in-flight executions cannot be deleted.

# Endpoint

`POST https://api.velt.dev/v2/workflow/definitions/delete`

# Headers

<ParamField header="x-velt-api-key" type="string" required>
  Your API key.
</ParamField>

<ParamField header="x-velt-auth-token" type="string" required>
  Your [Auth Token](/security/auth-tokens).
</ParamField>

# Body

#### Params

<ParamField body="data" type="object" required>
  <Expandable title="properties">
    <ParamField body="definitionId" type="string" required>
      The definition to delete.
    </ParamField>

    <ParamField body="organizationId" type="string">
      Required when scoped to an organization or document.
    </ParamField>

    <ParamField body="documentId" type="string">
      Required when scoped to a document.
    </ParamField>

    <ParamField body="purge" type="boolean" default="false">
      When `true`, hard-deletes the definition: the document and its entire version history are physically removed. When `false` (default), the definition is soft-deleted (tombstoned).
    </ParamField>
  </Expandable>
</ParamField>

## **Example Requests**

#### Soft-delete a definition (tombstone)

```JSON
{
  "data": {
    "definitionId": "marketing-copy-approval"
  }
}
```

#### Hard-delete a definition (purge version history)

```JSON
{
  "data": {
    "definitionId": "marketing-copy-approval",
    "purge": true
  }
}
```

# Response

The response includes a `purged` field reflecting which path ran: `false` for soft-delete, `true` for hard-delete.

#### Success Response

```JSON
{
  "result": {
    "deleted": true,
    "purged": false,
    "definitionId": "marketing-copy-approval"
  }
}
```

#### Failure Response

```JSON
{
  "error": {
    "message": "ERROR_MESSAGE",
    "status": "FAILED_PRECONDITION"
  }
}
```

**Errors:** `NOT_FOUND` (definition does not exist) / `FAILED_PRECONDITION` (in-flight executions exist).
