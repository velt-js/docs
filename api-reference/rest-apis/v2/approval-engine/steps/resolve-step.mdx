---
title: "Resolve Step"
api: "POST https://api.velt.dev/v2/workflow/steps/resolve"
---

Use this API to admin-override a step into a terminal `completed` state with an admin-supplied output. The step's aggregator is skipped; downstream edges fire as if the step had completed normally.

# Endpoint

`POST https://api.velt.dev/v2/workflow/steps/resolve`

# Headers

<ParamField header="x-velt-api-key" type="string" required>
  Your API key.
</ParamField>

<ParamField header="x-velt-auth-token" type="string" required>
  Your [Auth Token](/security/auth-tokens).
</ParamField>

# Body

#### Params

<ParamField body="data" type="object" required>
  <Expandable title="properties">
    <ParamField body="executionId" type="string" required>
      The execution containing the step.
    </ParamField>

    <ParamField body="stepId" type="string" required>
      The step to admin-override.
    </ParamField>

    <ParamField body="output" type="object">
      Admin-supplied output. Capped (each value ≤ 16k chars).
    </ParamField>

    <ParamField body="reason" type="string">
      ≤ 500 chars. Audit reason for the override.
    </ParamField>

    <ParamField body="actorId" type="string">
      ≤ 200 chars. Identifier of the administrator initiating the override.
    </ParamField>
  </Expandable>
</ParamField>

<Note>
Requires an admin-scoped auth token. Non-admin tokens are rejected with `PERMISSION_DENIED`.
</Note>

## **Example Requests**

#### Admin-override a stuck step

```JSON
{
  "data": {
    "executionId": "exec_1777374504255_xzy43k9q",
    "stepId": "step_agent-draft_..._lwofay__to__human-brand",
    "output": { "decision": "approve", "note": "approved by admin in emergency" },
    "reason": "Reviewer on PTO; emergency override",
    "actorId": "admin_jane"
  }
}
```

# Response

#### Success Response

```JSON
{
  "result": {
    "overridden": true,
    "stepId": "step_agent-draft_..._lwofay__to__human-brand"
  }
}
```

#### Failure Response

```JSON
{
  "error": {
    "message": "ERROR_MESSAGE",
    "status": "PERMISSION_DENIED"
  }
}
```

**Errors:** `FAILED_PRECONDITION` (step already terminal) / `PERMISSION_DENIED` / `NOT_FOUND`.
