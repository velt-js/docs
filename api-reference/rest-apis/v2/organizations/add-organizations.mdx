---
title: "Add Organizations"
api: "POST https://api.velt.dev/v2/organizations/add"
---

Use this API to add new organizations and its metadata.

# Endpoint

`POST https://api.velt.dev/v2/organizations/add`

# Headers

<ParamField header="x-velt-api-key" type="string" required>
  Your API key.
</ParamField>

<ParamField header="x-velt-auth-token" type="string" required>
  Your [Auth Token](/security/auth-tokens).
</ParamField>

# Body

#### Params

<ParamField body="data" type="object" required>
  <Expandable title="properties">
    <ParamField body="organizations" type="Organization[]" required>
      Array of Organization objects
    </ParamField>
  </Expandable>
</ParamField>

## **Example Requests**

#### Add organization

```JSON
{
  "data": {
    "organizations": [
      {
        "organizationId": "yourOrganizationId",
        "organizationName": "Your Organization Name"
      }
    ]
  }
}
```

# Response

#### Success Response

```JSON
{
  "result": {
    "status": "success",
    "message": "Organization(s) added successfully.",
    "data": {
      "yourOrganizationId": {
        "success": true,
        "id": "02cf91e5e7a5f4c0b600c84cf248384b",
        "message": "Added Successfully"
      }
    }
  }
}
```

#### Failure Response

For single validation errors:
```JSON
{
  "error": {
    "message": "Organization creation failed: organizationId is required",
    "status": "VALIDATION_FAILED",
    "details": {
      "field": "organizationId",
      "code": "MISSING_REQUIRED_FIELD"
    }
  }
}
```

For batch operations with partial failures (HTTP 500):
```JSON
{
  "error": {
    "message": "Some organizations failed to process",
    "status": "PARTIAL_FAILURE", 
    "data": {
      "successfulOrgId1": {
        "success": true,
        "id": "02cf91e5e7a5f4c0b600c84cf248384b",
        "message": "Added Successfully"
      },
      "failedOrgId2": {
        "success": false,
        "message": "Validation failed: organizationName is required",
        "errors": {
          "organizationName": ["Organization name cannot be empty"]
        }
      }
    }
  }
}
```

<Info>
**Batch Operation Error Handling**: When adding multiple organizations, if any organization fails validation or processing, the API returns HTTP status 500 with detailed information about which organizations were successfully created and which failed. This allows you to identify and retry only the failed operations.
</Info>

For other common errors, see the [Error Handling Guide](/api-reference/rest-apis/v2/error-handling).
