The Activity feature provides a real-time feed of collaboration events across your app. Activities are generated automatically for comments, reactions, recorder events, and CRDT edits, and you can also emit custom activities for non-Velt events (e.g., deployments, status changes).

## How It Works

- **Automatic generation**: Velt generates activity records whenever users interact with Comments, Reactions, Recorder, or CRDT features. For CRDT edits, use [`setActivityDebounceTime()`](/realtime-collaboration/crdt/setup/core#activity-debounce) to control how keystrokes are batched into activity records.
- **Custom activities**: Use `createActivity()` to push your own events into the same feed.
- **2-layer trigger listener architecture**: Activity records are created at the feature layer (e.g., a comment is added) and emitted through a subscription layer (`getAllActivities()`) that supports org-wide or scoped filtering.

## Use Cases

- **Org-wide activity feed**: Show every collaboration event across all documents in the organization.
- **Per-document feed**: Scope the feed to a single document and filter by feature type.
- **Custom events**: Merge application-level events (deployments, escalations) into the same activity stream.
