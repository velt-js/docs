## Disable ShadowDOM
By default, a ShadowDOM is used with certain components to ensure that your applicationâ€™s CSS does not interfere with the styling of the SDK components.

`Default: enabled`

If you want to customize the comments sidebar component using global CSS, you can do so by disabling the ShadowDOM.

<Tabs>

<Tab title="React / Next.js">
### Example
```jsx
<VeltCommentsSidebar shadowDom={false} />
```

### API methods

```jsx
const commentElement = client.getCommentElement();
commentElement.enableSidebarShadowDOM();
commentElement.disableSidebarShadowDOM();
```
</Tab>

<Tab title="HTML">
### Example
```jsx
<velt-comments-sidebar shadow-dom="false"></velt-comments-sidebar>
```

### API methods

```jsx
const commentElement = Velt.getCommentElement();
commentElement.enableSidebarShadowDOM();
commentElement.disableSidebarShadowDOM();
```
</Tab>

</Tabs>


## Dark Mode
![](/images/customization/velt-comment-sidebar-1.png)

![](/images/sidebar-breakdown-dark.png)
By default, all components are in Light Mode, but there are several properties and methods to enable Dark Mode.

`Default: false`

To enable Dark Mode for comments sidebar:
<Tabs>

<Tab title="React / Next.js">
### Example
```js
<VeltCommentsSidebar darkMode={true} />
```

### API methods

```jsx
const commentElement = client.getCommentElement();
commentElement.enableDarkMode();
commentElement.disableDarkMode();
```

</Tab>

<Tab title="HTML">
### Example
```js
<velt-comments-sidebar dark-mode="true"></velt-comments-sidebar>
```

### API methods

```jsx
const commentElement = Velt.getCommentElement();
commentElement.enableDarkMode();
commentElement.disableDarkMode();
```
</Tab>
</Tabs>
