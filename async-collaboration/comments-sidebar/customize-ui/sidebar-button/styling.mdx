## Disable ShadowDOM
By default, a ShadowDOM is used with certain components to ensure that your applicationâ€™s CSS does not interfere with the styling of the SDK components.

`Default: enabled`

If you want to customize the sidebar button component using global CSS, you can do so by disabling the ShadowDOM.

<Tabs>

<Tab title="React / Next.js">
### Example
```jsx
<VeltSidebarButton shadowDom={false} />
```

</Tab>

<Tab title="HTML">
### Example
```jsx
<velt-sidebar-button shadow-dom="false"></velt-sidebar-button>
```
</Tab>
</Tabs>
## Dark Mode
![](/images/sidebar-button-dark.png)

By default, all components are in Light Mode, but there are several properties and methods to enable Dark Mode.

`Default: false`

To enable Dark Mode for sidebar button:
<Tabs>

<Tab title="React / Next.js">
### Example
```js
<VeltSidebarButton darkMode={true} />
```

### API methods

```jsx
const commentElement = client.getCommentElement();
commentElement.enableDarkMode();
commentElement.disableDarkMode();
```

</Tab>

<Tab title="HTML">
### Example
```js
<velt-sidebar-button dark-mode="true"></velt-sidebar-button>
```

### API methods

```jsx
const commentElement = Velt.getCommentElement();
commentElement.enableDarkMode();
commentElement.disableDarkMode();
```

</Tab>
</Tabs>
