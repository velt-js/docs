## Disable ShadowDOM
By default, a ShadowDOM is used with certain components to ensure that your applicationâ€™s CSS does not interfere with the styling of the SDK components.

`Default: enabled`

If you want to customize the comment bubble component using global CSS, you can do so by disabling the ShadowDOM.

<Tabs>

<Tab title="React / Next.js">


### Example
```jsx
<VeltCommentBubble shadowDom={false} />
```
</Tab>
<Tab title="HTML">
### Example
```jsx
<velt-comment-bubble shadow-dom="false"></velt-comment-bubble>
```
</Tab>
</Tabs>
## Dark Mode

![](/images/customization/velt-comment-bubble-1.png)

By default, all components are in Light Mode, but there are several properties and methods to enable Dark Mode.

`Default: false`

Below are examples to enable dark mode for comment bubble component.

<Tabs>

<Tab title="React / Next.js">
### Example
```js
<VeltCommentBubble darkMode={true} />
```

### API methods

```jsx
const commentElement = client.getCommentElement();
commentElement.enableDarkMode();
commentElement.disableDarkMode();
```

</Tab>

<Tab title="HTML">

### Example
```HTML
<velt-comment-bubble dark-mode="true"></velt-comment-bubble>
```

### API methods

```jsx
const commentElement = Velt.getCommentElement();
commentElement.enableDarkMode();
commentElement.disableDarkMode();
```

</Tab>

</Tabs>
