## REST APIs MCP Server

The Velt REST API MCP Server allows you to interact with your Velt data directly from various MCP (Multi-Code-Project) compatible clients, such as Cursor and Claude Desktop. This enables you to leverage Velt's collaborative features within your development environment.

### Features

The MCP server exposes the full capabilities of the Velt REST API, including managing:

*   [Organizations](/api-reference/rest-apis/organizations/get-organizations-v2)
*   [Folders](/api-reference/rest-apis/folders/get-folders)
*   [Documents](/api-reference/rest-apis/documents/get-documents-v2)
*   [Users](/api-reference/rest-apis/users/get-users-v2)
*   [User Groups](/api-reference/rest-apis/organization-user-groups/add-groups)
*   [Comments](/api-reference/rest-apis/comments-feature/comments/get-comments)
*   [Notifications](/api-reference/rest-apis/notifications/get-notifications-v2)
*   [Live State](/api-reference/rest-apis/livestate/broadcast-event)
*   [Workspace](/api-reference/rest-apis/workspace/add-domain)

### Setup

You can set up the REST API MCP Server using one of the following methods.

#### Using Package Installer

The easiest way to get started is by using the `@smithery/cli` package installer. This will automatically download and configure the MCP server for your chosen client.

For example, to install it for Claude Desktop:

```bash
npx -y @smithery/cli install @velt-js/rest-api-mcp --client claude
```

After installation, you may also need to install the Velt API MCP package in your project:

```bash
npm i @veltdev/api-mcp
```

#### Manual Configuration

If you prefer to configure the server manually, follow these steps.

1.  **Download the server:**
    Clone the Velt MCP Server repository or download the source code as a ZIP file. You will need to build the project if you clone it, or use the pre-built files in the `dist` directory.

2.  **Configure your client:**

    ##### Claude Desktop
    1.  Open Claude Desktop.
    2.  Go to **Settings > Developer > Edit Config**.
    3.  Add the following configuration to the `mcpServers` section in your config file. Make sure to replace the placeholder values.

        ```json
        {
          "mcpServers": {
            "velt-api-mcp": {
              "command": "node",
              "args": ["<path-to-unzip-velt-api-mcp-server-folder>/dist/index.js"],
              "env": {
                "VELT_API_KEY": "<your-velt-api-key>",
                "VELT_AUTH_TOKEN": "<your-velt-auth-token>"
              }
            }
          }
        }
        ```

    4.  Save the configuration file.
    5.  Restart Claude Desktop. The `velt-api-mcp` server will now be available in the tools menu.

    ##### Cursor
    1.  Open Cursor.
    2.  Go to **File → Cursor Settings → MCP → Add new server**.
    3.  Add the following configuration, replacing the placeholders with the full path to the server and your Velt secrets.

        ```json
        {
          "mcpServers": {
            "velt-api-mcp": {
              "command": "node",
              "args": ["<path-to-unzip-velt-api-mcp-server-folder>/dist/index.js"],
              "env": {
                "VELT_API_KEY": "<your-velt-api-key>",
                "VELT_AUTH_TOKEN": "<your-velt-auth-token>"
              }
            }
          }
        }
        ```
    4.  Ensure the server is enabled in the MCP menu.

### Example Usage

Once configured, you can interact with the Velt API through your MCP client. For example, you can query for documents or add comments programmatically from your editor. The following examples demonstrate how to use the `velt-api-mcp` tool in a JavaScript environment.

#### Get all documents in an organization

This example shows how to retrieve all documents for a given organization.

```javascript
// Assuming 'veltApiMcp' is the object to interact with the MCP server
async function getAllDocuments(organizationId) {
  try {
    const response = await veltApiMcp.documents.get({
      organizationId: organizationId,
    });
    console.log('Successfully retrieved documents:', response.data);
    return response.data;
  } catch (error) {
    console.error('Failed to get documents:', error);
  }
}

// Usage
const documents = await getAllDocuments('your-organization-id');
```

#### Add a comment to a document

This example demonstrates how to add a comment to a specific document and annotation.

```javascript
// Assuming 'veltApiMcp' is the object to interact with the MCP server
async function addComment(data) {
  const { organizationId, documentId, annotationId, userId, commentText } = data;
  try {
    const response = await veltApiMcp.comments.add({
      organizationId,
      documentId,
      annotationId,
      commentData: [
        {
          commentText,
          from: {
            userId,
          },
        },
      ],
    });
    console.log('Successfully added comment:', response);
    return response;
  } catch (error) {
    console.error('Failed to add comment:', error);
  }
}

// Usage
await addComment({
  organizationId: 'your-organization-id',
  documentId: 'your-document-id',
  annotationId: 'your-annotation-id',
  userId: 'user-who-comments',
  commentText: 'This is a test comment from the MCP server.',
});
```

For more details on the available endpoints and their parameters, please refer to the [REST API documentation](/api-reference/rest-apis).

## Docs MCP Server

The Velt Docs MCP Server allows you to search and browse Velt's documentation directly within your MCP-compatible client, like Cursor, Windsurf, Claude Desktop and more. This enables you to quickly find information about our SDKs, APIs, and features without having to leave your editor, streamlining your development workflow when implementing Velt.

### Setup
```npx mint-mcp add velt```

### Example Usage

Once the Velt Docs MCP Server is configured, you can use it to search for documentation from your client.

For example, in Cursor, you can use the `@velt` command in the chat to ask questions about Velt.

**Query:**
`@velt how to customize comment pins`

**Result:**
The MCP will return relevant documentation pages and snippets related to customizing comment pins, allowing you to quickly access the information you need.
