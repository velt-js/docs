## Detect if Velt SDK is initialized

Sometimes you may need to detect if the Velt SDK is initialized. You can do this by using the following.

<Tabs>

<Tab title="React / Next.js">

**Using Hooks:**
```jsx
const veltInitState = useVeltInitState();
useEffect(() => {
    console.log('Velt Init State:', veltInitState);
    if (veltInitState) {
        // Velt state is initialized, so user can perform any action here
    }
}, [veltInitState]);
```

**Using API:**
```jsx
client.getVeltInitState().subscribe((veltInitState: boolean | undefined) => {
	console.log('Velt Init State:', veltInitState);
});
```

</Tab>
  
  
<Tab title="Other Frameworks">


```jsx
Velt.getVeltInitState().subscribe((veltInitState: boolean | undefined) => {
	console.log('Velt Init State:', veltInitState);
});
```

</Tab>
</Tabs>


## Get the current user
You may need to get the current user to see if all the fields like userId, organizationId etc are correctly being set.

<Tabs>

<Tab title="React / Next.js">

**Using API:**
```jsx
// Use this in your code
const user = client.getUser();

// Or, use this in the browser console:
const user = Velt.getUser();
```

</Tab>
  
  
<Tab title="Other Frameworks">
```jsx
// Use this in your code or in the browser console
const user = Velt.getUser();
```
</Tab>
</Tabs>


## Get the current organization, document and location
You may need to verify the current organization, document and location set to see if all the structural entities are correctly being set.

<Tabs>

<Tab title="React / Next.js">
```jsx
// Use this in your code
const { organization, documents, locations } = await client.getMetadata();

// Or, use this in the browser console:
const { organization, documents, locations } = await Velt.getMetadata();
```

</Tab>
  
  
<Tab title="Other Frameworks">
```jsx
// Use this in your code or in the browser console
const { organization, documents, locations } = await Velt.getMetadata();
```
</Tab>
</Tabs>

## Check the current Velt SDK version
Sometimes the browser or the local npm modules might still be loading the older version of the Velt SDK. You can check the current Velt SDK version by using the following.
```javascript
// Use this in the browser console:
Velt.version;
```


## Debugging Tools

### Velt Console Debugger
Monitor real-time events from active sessions.

<Steps>
  <Step title="Open Debugger">
    Go to [Velt Console Debugger](https://console.velt.dev/dashboard/debugger)
  </Step>
  <Step title="Start Monitoring">
    Click the `Play` button
  </Step>
  <Step title="Filter Events">
    Use filters to focus on specific event types or users
  </Step>
</Steps>

### Browser Dev Console
Monitor SDK events directly in your application. This is an alternative to the Velt Console Debugger.


<Steps>
  <Step title="Open Network Tab">
    Open browser dev tools (F12) and go to Network tab
  </Step>
  <Step title="Filter Requests">
    Filter by `velt`
  </Step>
  <Step title="Find Events">
    Look for event name: `sa` to see SDK events. These are the same events you will find in the debugger in the Velt Console.
  </Step>
</Steps>

## Common Issues and Solutions

<AccordionGroup>
  <Accordion title="SDK Not Initializing">
    **Check:**
    - API key is correct
    - Domain is safelisted in Velt console
    - `VeltProvider` is at the root of your app (react)
    - 'use client' is added at the top of the file (nextjs)
    - Browser console for errors
  </Accordion>
  
  <Accordion title="User Not Identified">
    **Check:**
    - `identify` is called with valid user object
    - User object contains `userId` and `organizationId` fields
    - Component is a child of `VeltProvider` (react)
  </Accordion>
  
  <Accordion title="Comment Tool not working">
    **Check:**
    - User is identified
    - Document ID is being set correctly
  </Accordion>
</AccordionGroup>
