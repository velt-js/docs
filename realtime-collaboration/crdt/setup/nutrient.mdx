Enable real-time collaborative Nutrient Web SDK document state, annotations, comments, form values, and selections. The integration stores Nutrient Instant JSON in a Velt CRDT Store backed by [Yjs](https://docs.yjs.dev/).

## Prerequisites

- Node.js (v14 or higher)
- React (v18 or higher)
- A Nutrient Web SDK license
- A Velt account with an API key ([sign up](https://console.velt.dev/))
- Optional: TypeScript for type safety

## Setup

### Step 1: Install Dependencies

```bash
npm install @veltdev/nutrient-crdt-react @veltdev/nutrient-crdt @nutrient-sdk/viewer @veltdev/react @veltdev/types
```

Keep one copy of React, `@veltdev/react`, and `@veltdev/nutrient-crdt` in the application bundle.


### Step 2: Setup Velt

Initialize Velt with server-backed authentication by following the [Velt Setup Docs](/get-started/quickstart), then set a stable document before the viewer initializes:

```tsx
import { useSetDocument } from '@veltdev/react';

useSetDocument('nutrient-review-123', { documentName: 'Nutrient review' });
```

### Step 4: Customize Viewer Creation (Optional)

The component accepts a Nutrient document input, license, CDN choice, viewer options, a module override, or a custom factory:

```tsx
import NutrientViewer from '@nutrient-sdk/viewer';

<NutrientCrdtEditor
  documentId="nutrient-review-123"
  document={documentUrl}
  useCDN
  viewerOptions={{ toolbarItems }}
  createInstance={async (host, initialContent) => {
    return NutrientViewer.load({
      container: host,
      document: documentUrl,
      instantJSON: initialContent,
      licenseKey: 'YOUR_NUTRIENT_LICENSE_KEY',
      useCDN: true,
    });
  }}
/>
```

Pass `instance` when the application already owns the viewer; the component will not load or unload that external instance.


### Step 5: Status Monitoring (Optional)

```tsx
const {
  manager,
  status,
  synced,
  isLoading,
  error,
  stats,
} = collaboration;

useEffect(() => {
  if (!manager) return;
  const offStatus = manager.onStatusChange(console.log);
  const offSynced = manager.onSynced(console.log);
  return () => {
    offStatus();
    offSynced();
  };
}, [manager]);
```

Local viewer actions can continue while reconnecting. `stats` reports viewer and Store availability, remote selection count, and last flush reason.


### Step 6: Version Management (Optional)

```tsx
const versionId = await collaboration.saveVersion('Before approval');
const version = await collaboration.getVersionById(versionId);
await collaboration.refreshVersions();
await collaboration.restoreVersion(versionId);
```

The manager flushes the viewer's current Instant JSON before saving. Restoring reapplies the saved state to the attached viewer.


### Step 7: Initial Content and Force Reset (Optional)

Use Nutrient Instant JSON as seed content:

```tsx
const initialContent = {
  format: INSTANT_JSON_FORMAT,
  annotations: [],
  comments: [],
  formFieldValues: { customer: 'Acme Corp' },
};
```

The seed is used only when the shared Store has no document state. For an explicit reset:

```tsx
await collaboration.forceReset(initialContent);
```

<Warning>`forceResetInitialContent` and `forceReset()` replace shared annotations, comments, and form values for every collaborator.</Warning>


### Step 8: Apply Document Changes

Prefer focused helpers for ordinary application actions:

```tsx
await collaboration.setFormFieldValue('customer', 'Dunder Mifflin');
await collaboration.addHighlight({
  pageIndex: 0,
  bbox: [72, 150, 220, 30],
  color: '#fff59d',
  text: 'Payment terms',
});
await collaboration.addComment({
  text: 'Needs legal review',
  pageIndex: 0,
});
```

Use full snapshot methods for imports or specialized workflows:

```tsx
await collaboration.applyInstantJson(nextInstantJson, 'import');
await collaboration.flushInstanceToStore('manual-save', true);
collaboration.refreshSnapshots();
```


### Step 9: Configure Remote Selections (Optional)

```tsx
collaboration.setLocalSelection({
  pageIndex: 0,
  rects: [{ left: 72, top: 150, width: 220, height: 30, pageIndex: 0 }],
  text: 'Selected text',
});

collaboration.renderRemoteSelections();
```

`remoteSelections` is reactive awareness state. It is not persisted as Instant JSON or included in versions. Call `clearRemoteSelectionOverlays()` before replacing the viewer or overlay host.


### Step 10: CRDT Event Subscription (Optional)

Subscribe to normalized document changes through the manager:

```tsx
useEffect(() => {
  if (!collaboration.manager) return;
  return collaboration.manager.onDocumentChange((event) => {
    console.log(event.source, event.reason, event.state);

```

For application-wide Velt events:

```tsx
const subscription = client.getCrdtElement().on('updateData').subscribe(console.log);
```

### Step 14: Attach, Detach, and Cleanup

Initialize collaboration before the viewer exists when necessary:

```tsx
const collaboration = useCollaboration({
  editorId: 'nutrient-review-123',
  initializeWithoutInstance: true,
});

collaboration.attachInstance(instance, { applyRemoteState: true });
collaboration.detachInstance();
```

The hook destroys its manager automatically. The drop-in component unloads only viewers it created. Call `destroy()` for early collaboration teardown.


## Notes

- **Snapshot model**: Nutrient Instant JSON is synchronized as document snapshots, not as character-level PDF edits.
- **Document scope**: The adapter persists annotations, comments, and form field values; the PDF input remains a viewer resource.
- **Viewer capability**: Full persistence requires an instance that supports Instant JSON export and application operations.
- **Awareness**: Remote selections are transient and not part of versioned document content.
- **Debounced flushes**: Viewer changes are batched before Store writes; default debounce is 120 ms.
- **Unique editorId**: Collaborators must share the same Velt document and editor ID.
- **Production authentication**: Use a server-backed Velt `authProvider`; never expose privileged server tokens in browser code.


## Testing and Debugging

**To test collaboration:**
1. Open the same PDF, Velt document, and Nutrient `editorId` as two different users.
2. Add annotations, comments, highlights, and form values in both clients.
3. Verify remote selections, version restore, reconnection, and persistence after reload.
4. Test viewer unload/reload and an attach-later flow.

**Common issues:**
- Viewer not loading: Verify the document URL, Nutrient license, CDN choice, and host height.
- Collaboration remains loading: Confirm Velt authentication and a viewer instance, or use `initializeWithoutInstance`.
- Missing document changes: Confirm the viewer supports `exportInstantJSON` and relevant change events.
- Overlay position is incorrect: Render into the correct viewer host and refresh after viewport changes.
- Existing data resets: Disable `forceResetInitialContent` during normal loads.

<Tip>
Use the [VeltCrdtStoreMap debugging interface](/realtime-collaboration/crdt/setup/core#veltcrdtstoremap) to inspect the Instant JSON state and provider status.

## Complete Example

```tsx Complete App.tsx expandable lines
import { useMemo, useState } from 'react';
import { VeltProvider, useSetDocument } from '@veltdev/react';
import {
  INSTANT_JSON_FORMAT,
  NutrientCrdtEditor,
  type CollaborationManager,
  type InstantJson,
} from '@veltdev/nutrient-crdt-react';

const DOCUMENT_ID = 'nutrient-review-123';
const USER = {
  userId: 'ada',
  organizationId: 'my-org-id',
  name: 'Ada Lovelace',
  email: 'ada@example.com',
};

const initialContent: InstantJson = {
  format: INSTANT_JSON_FORMAT,
  annotations: [],
  comments: [],
  formFieldValues: { customer: 'Acme Corp' },
};

function CollaborativeDocument() {
  const [manager, setManager] = useState<CollaborationManager | null>(null);
  const [versionName, setVersionName] = useState('');

  useSetDocument(DOCUMENT_ID, { documentName: 'Nutrient review' });

  const saveVersion = async () => {
    if (!manager || !versionName.trim()) return;
    await manager.saveVersion(versionName.trim());
    setVersionName('');
  };

  return (
    <main>
      <input value={versionName} onChange={(event) => setVersionName(event.target.value)} />
      <button onClick={saveVersion}>Save version</button>
      <NutrientCrdtEditor
        documentId={DOCUMENT_ID}
        document="/documents/contract.pdf"
        licenseKey="YOUR_NUTRIENT_LICENSE_KEY"
        initialContent={initialContent}
        className="nutrient-host"
        onManagerReady={setManager}
        onStatusChange={(status) => console.log('Status:', status)}
        onError={console.error}
      >
        {(collaboration) => (
          <p>{collaboration.status} · {collaboration.synced ? 'Synced' : 'Syncing'}</p>
        )}
      </NutrientCrdtEditor>
    </main>
  );
}

export default function App() {
  const authProvider = useMemo(() => ({
    user: USER,
    retryConfig: { retryCount: 3, retryDelay: 1000 },
    generateToken: async () => {
      const response = await fetch('/api/velt/token', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(USER),
      });
      const { token } = await response.json();
      return token;
    },
  }), []);

  return (
    <VeltProvider apiKey="YOUR_VELT_API_KEY" authProvider={authProvider}>
      <CollaborativeDocument />
    </VeltProvider>
  );
}
```

## How It Works

1. The wrapper waits for Velt, an authenticated user, an editor ID, and a Nutrient instance when required.
2. The manager normalizes Instant JSON into a versioned `NutrientDocumentState`.
3. Viewer change events schedule debounced snapshot exports to the Velt CRDT Store.
4. Remote document state is applied to the attached Nutrient instance.
5. Awareness carries page rectangles and selected text for transient remote overlays.
6. Versions save and restore the complete Instant JSON state.
