# 3.0.78

### Features
- [**Velt Button**]: Added Velt Button Wireframe Component with support for various types ('button', 'button-toggle', 'multi-select', 'single-select'), disabled state, active state, and group functionality. Available in React and HTML with corresponding event handling APIs.

### Improvements
- [**Comments**]: Added missing wireframes for Comment Dialog Toggle Reply children.
- [**Comments**]: Now users can't submit empty comments during editing.
- [**Ergonomics**]: Updated `useCommentActionCallback` to `useCommentEventCallback`. The old hook name will continue to work.
- [**Ergonomics**]: For Sidebar Custom actions and filters, updated APIs and Hooks:
  - `onCommentSidebarInit` can now be accessed with `useCommentEventCallback('commentSidebarDataInit')` or `commentElement.on('commentSidebarDataInit')`
  - `onCommentSidebarData` can now be accessed with `useCommentEventCallback('commentSidebarDataUpdate')` or commentElement.on('commentSidebarDataUpdate')`
  - `onCommentSidebarActionButtonClick` can now be accessed with `useCommentEventCallback('commentSidebarActionButtonClick')`

### Bug Fixes
- [**Comments**]: Fixed an issue where the Comment sidebar group count was not updated correctly.
- [**Comments**]: Fixed an issue where attached files were not displayed on the very first comment in inline and page mode comments.
- [**Comments**]: Fixed a UI issue where attachments were misaligned when a comment was edited.
- [**Comments**]: Fixed a UX issue where the comment composer did not scroll into view when comment edit button was clicked.
- [**Comments**]: Resolved a UI issue where the Page Mode composer in the comment sidebar during embed mode was not taking the full width of the parent container.
- [**Comments**]: Fixed padding issue with the @mention autocomplete dropdown.
