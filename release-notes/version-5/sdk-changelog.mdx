### Breaking Changes

- [**Comments — Sidebar V2**]: The `minimal-actions-dropdown-v2` component family has been **removed** from Comment Sidebar V2 and replaced by the new unified `actions`-type dropdown in `minimal-filters`. Removed web components: `velt-comment-sidebar-minimal-actions-dropdown-v2` (plus `-trigger-v2`, `-content-v2`, `-content-mark-all-read-v2`, `-content-mark-all-resolved-v2`). Removed React components: `VeltCommentSidebarV2MinimalActionsDropdown` and its sub-exports, plus the `VeltCommentsSidebarV2Wireframe.MinimalActionsDropdown` slot.

  **Migration:**

<Tabs>
<Tab title="React / Next.js">
```tsx
// Before (removed)
<VeltCommentsSidebarV2Wireframe>
  <VeltCommentsSidebarV2Wireframe.MinimalActionsDropdown>
    {/* mark all read / mark all resolved */}
  </VeltCommentsSidebarV2Wireframe.MinimalActionsDropdown>
</VeltCommentsSidebarV2Wireframe>

// After — configure via minimal-filters
<VeltCommentsSidebarV2 minimalFilters={[{ type: 'actions' }]} />
```
</Tab>
<Tab title="Other Frameworks">
```html
<!-- Before (removed) -->
<velt-comment-sidebar-minimal-actions-dropdown-v2></velt-comment-sidebar-minimal-actions-dropdown-v2>

<!-- After -->
<velt-comments-sidebar-v2 minimal-filters='[{"type":"actions"}]'></velt-comments-sidebar-v2>
```
</Tab>
</Tabs>

  Apps that did not customize the minimal-actions dropdown require no changes.

- [**Comments — Sidebar V2**]: The React `position` prop on `VeltCommentsSidebarV2` has been narrowed from `string` to `'right' | 'left'`. Callers passing other string values will now get a TypeScript error.

### New Features

- [**Comments — Sidebar V2**]: Complete declarative **filtering, sorting, grouping, and search** system added to `<velt-comments-sidebar-v2>` / `VeltCommentsSidebarV2`. Configure which field-filter panels appear (`filters`, `mini-filters`), add multiple configurable header dropdowns (`minimal-filters` with `type: 'filter' | 'sort' | 'quick' | 'actions'`), control how filters combine across fields (`filter-operator`), set a default sort (`sort-by` / `sort-order`) and quick filter, enable collapsible grouping (`group-config`), and add a free-text search box — all as declarative props/attributes.

  Key new props:

  | Prop (React) | Attribute (HTML) | Type | Default |
  |---|---|---|---|
  | `filters` | `filters` | `FilterField[] \| object` | `[]` |
  | `miniFilters` | `mini-filters` | `FilterField[]` | `[]` |
  | `minimalFilters` | `minimal-filters` | `SidebarMinimalFilterConfig[]` | `[]` |
  | `filterOperator` | `filter-operator` | `'and' \| 'or'` | `'and'` |
  | `filterPanelLayout` | `filter-panel-layout` | `'bottomSheet' \| 'menu'` | `'bottomSheet'` |
  | `filterOptionLayout` | `filter-option-layout` | `'dropdown' \| 'checkbox'` | `'dropdown'` |
  | `filterCount` | `filter-count` | `boolean` | `true` |
  | `sortBy` | `sort-by` | `SortBy` | — |
  | `sortOrder` | `sort-order` | `SortOrder` | — |
  | `defaultMinimalFilter` | `default-minimal-filter` | `'all' \| 'read' \| 'unread' \| 'resolved' \| 'open' \| 'assignedToMe' \| 'reset'` | — |
  | `groupConfig` | `group-config` | `{ enable?: boolean; name?: string; groupBy?: string }` | — |
  | `fullScreen` | `full-screen` | `boolean` | `false` |
  | `fullExpanded` | `full-expanded` | `boolean` | `false` |
  | `shadowDom` | `shadow-dom` | `boolean` | on |
  | `searchPlaceholder` | `search-placeholder` | `string` | `'Search comments'` |
  | `sidebarButtonCountType` | `sidebar-button-count-type` | `'default' \| 'filter'` | — |

  Built-in filter field IDs: `status`, `priority`, `category`, `people`, `assigned`, `tagged`, `involved`, `location`, `version`, `document`. Within a single field, selections combine with OR; `filter-operator` controls how fields combine.

  New event: `onFullscreenClick` — fires when the fullscreen toggle is clicked.

- [**Comments — Sidebar V2**]: New primitive components for the filter UI, search box, list group header, and fullscreen button — available as web components, React components, and wireframe slots.

  **Filter button:** `velt-comment-sidebar-filter-button-v2` / `VeltCommentSidebarV2FilterButton` (+ applied-icon sub-element).

  **Search:** `velt-comment-sidebar-search-v2` (+ `-icon`, `-input`) / `VeltCommentSidebarV2Search*`.

  **List group header:** `velt-comment-sidebar-list-group-header-v2` (+ `-label`, `-count`, `-chevron`, `-separator`) / `VeltCommentSidebarV2ListGroupHeader*`.

  **Fullscreen button:** `velt-comment-sidebar-fullscreen-button-v2` / `VeltCommentSidebarV2FullscreenButton`.

  **Filter container (Main Filter bottom-sheet — entirely new):** `velt-comment-sidebar-filter-container-v2` and 20+ sub-elements covering the container title, close/apply/reset buttons, group-by selector, and the full section → field → control → option tree. React equivalents: `VeltCommentSidebarV2FilterContainer*`.

  **Filter dropdown additions:** `velt-comment-sidebar-filter-dropdown-content-list-category-label-v2` and `velt-comment-sidebar-filter-dropdown-content-list-item-count-v2` (React: `VeltCommentSidebarV2FilterDropdownContentListCategoryLabel`, `VeltCommentSidebarV2FilterDropdownContentListItemCount`).

- [**Self-Hosting**]: Fail-closed PII allowlist for self-hosting writes to the Velt DB. When a resolver is configured for comments, reactions, recordings, or activity data, the record persisted to the Velt DB now contains **only explicitly allowlisted, non-PII fields**. Known-PII fields (`commentText`, `commentHtml`, `targetTextRange.text`, nested `User` fields) are dropped from the Velt DB write. Open-typed customer-owned fields (`metadata`, `context`, `location`, `customList`) pass through unchanged. Without a resolver configured, behavior is unchanged. The data sent to the customer's own resolver is never filtered.

### API Changes

- [**Comments**]: New public method `CommentService.applyCommentSidebarClientFilters()` applies `CommentSidebarFilters` to a set of annotations honoring the current `systemFiltersOperator`. Backs the declarative `filters` object-shape input on the V2 sidebar. Additive — no existing call sites affected.

  ```ts
  public applyCommentSidebarClientFilters(
    annotations: CommentAnnotation[],
    filters: CommentSidebarFilters
  ): CommentAnnotation[];
  ```

- [**Comments — Sidebar V2**]: New exported types: `FilterField`, `BuiltInFilterFieldId`, `SidebarMinimalFilterConfig`, `SidebarSortConfig`, `SidebarSortCriterion`, `SidebarQuickFilterConfig`, `SidebarQuickCondition`, `SidebarQuickPredicate`, `SidebarAnnotationGroup`, `SidebarListRow`, `FilterFieldOperator`, `SidebarFilterDropdownType`, `FacetContext`, `FilterFieldResolver`, `SectionControlChip`, `SectionAllOption`, and the `BUILT_IN_FILTER_FIELD_IDS` constant. Approximately 60 new React prop interfaces for sidebar V2 primitives and wireframe variants. All additive.

### Bug Fixes

- [**Comments — Sidebar V2**]: Fixed grouping not activating when `groupConfig` is `{ enable: true }` without an explicit `groupBy`. The sidebar now defaults to grouping by `'location'` (page) when only `enable: true` is set, matching V1 behavior. Explicit `groupBy` values are still honored; `{ enable: false }` still flattens the list.

- [**Comments — Sidebar V2**]: Fixed wireframe overrides being silently dropped when a scoped dialog wireframe slot was rendered inside the sidebar's virtual-scroll list. The wireframe host was not yet attached to the DOM when `processWireframe()` was first called; processing is now deferred to `ngAfterViewInit`.

</Update>

<Update label="5.0.2-beta.32" description="June 4, 2026">

### New Features

- [**Comments**]: A new Agent Suggestion card renders for AI-agent suggestions — annotations where `annotation.agent` is present and `annotation.type` is `'suggestion'`. It shows an agent-identity header (name, avatar with fallback icon, relative timestamp), a bold title from `annotation.agent.result.title` plus the comment body, accept/reject buttons, and a 3-dot overflow menu — with a truncated body and an "Open Comment" footer link in sidebar mode. The card is admin-only (`appState.isUserAdmin`).
  - Accept button flips `suggestion.status` to `accepted`, sets `annotation.type = 'comment'`, marks the thread terminal, and fires `suggestionAccepted`;
  - Reject button flips the status to `rejected` and fires `suggestionRejected`. Both are idempotent (no-op unless status is `pending`) and return `null` on error without mutating state.

</Update>

<Update label="5.0.2-beta.31" description="May 22, 2026">

### New Features

- [**Approval Engine**]: Added an inbound webhook endpoint (`approvalwebhookhandler`) that accepts raw JSON POSTs from third-party systems (e.g. Salesforce, Stripe, GitHub) without requiring the `{data: ...}` envelope. Includes bearer-token validation, rate limiting, body-size limits, and SSRF URL guards. [Learn more →](/ai/approval-engine/overview#inbound-webhook-handler)

- [**Agents**]: New `listAgentExecutions` endpoint returns paginated agent execution history (`pageSize` + `cursor`), so you can page through an agent's prior runs without per-execution follow-up calls. [Learn more →](/api-reference/rest-apis/v2/agents/list-agent-executions)

- [**Approval Engine**]: Added a `__mock__` synthetic-agent escape hatch for agent nodes, letting tests and dry-runs exercise agent-node fixtures deterministically without invoking real agent infrastructure. [Learn more →](/ai/approval-engine/setup)

- [**Memory**]: Added a knowledge-base vector search endpoint (`POST /v2/memory/knowledge/search`): cosine nearest-neighbour search over your ingested knowledge content, distinct from the learned judgments that `search`/`ask` read. Pass a single `sourceId` to restrict to one source, an array of 2 to 30 ids to fan out and merge by distance, or omit it to search the whole workspace. [Learn more →](/api-reference/rest-apis/v2/memory/knowledge/search)

- [**Memory**]: Added XLSX multi-sheet ingestion. Ingested spreadsheets now capture content across all sheets, and the normalize-to-markdown step preserves every source heading. [Learn more →](/ai/memory/setup)

- [**Memory**]: Comment data from older SDK clients is now enriched into Memory: live comment enrichment plus a one-time backfill of historical data across eligible workspaces. [Learn more →](/ai/memory/overview#where-the-data-comes-from)

### Improvements

- [**Approval Engine**]: Approval Engine reads and writes are now routed through the per-tenant `storeDbId`, enforcing correct data isolation for customers on dedicated databases. [Learn more →](/ai/approval-engine/overview#scope)

- [**Memory**]: Knowledge-base searches are now vector-indexed for both whole-workspace and source-narrowed queries, so `knowledge/search` stays fast and correctly filtered at either scope.

- [**Memory**]: `knowledge/search` is workspace-scoped by design: `organizationId`/`documentId` are rejected by the strict schema rather than accepted and ignored, so the API contract accurately reflects what the endpoint filters on.

### Bug Fixes

- [**Approval Engine**]: Concurrent test runs of the same workflow now use fully isolated per-run IDs, preventing collisions when multiple test runs execute at the same time.

- [**Agents**]: `listAgentExecutions` responses now include `status` and a human-readable `message` per execution, so a single call enumerates execution outcomes without extra per-execution round trips.

- [**Approval Engine**]: Hardened production behavior: webhook error classification, recovery of stranded executions, loop exit and failure handling, retry-safe scheduling, and approval-action routing fixes.

- [**API**]: Restored the `for <path>` field-path suffix on `invalid_type` validation errors. The suffix now lives in the structured error payload instead of `error.message`, so clients that read the field path to surface field-level UI errors keep working.

- [**Video Player**]: Fixed comment and reaction pins on the video timeline: clicking a pin now opens the annotation instead of making it disappear, and adding a reaction at a position that already has one updates the existing reaction instead of saving a duplicate. The fixes also apply to annotations created before this release.

- [**Memory**]: Knowledge chunks from multiple sources now sort stably and predictably when scores are equal or missing, and XLSX per-sheet coverage is detected accurately via exact heading matching.

</Update>
