### New Features

- [**Access Control**]: `setPermissionProvider` now accepts a `dev: true` flag that routes permission checks through the browser instead of the server-to-server Real-Time Permission Provider (RTPP). This lets you test RTPP against a `localhost` endpoint without a public tunnel (no ngrok or Cloudflare Tunnel needed). Supply `endpointConfig.url` to POST to a local URL, or `resolvePermissions` to resolve inline. Opt-in only â€” no change to default behavior. Production API keys ignore the flag server-side. **Prerequisite:** The Real-Time Permission Provider must be enabled in the [Velt Console](https://console.velt.dev). [Learn more â†’](/key-concepts/overview#c-real-time-permission-provider)

### Improvements

- [**Comments**]: Agent suggestion comments and accept/reject actions are now visible and actionable by all viewers, not just admins. The `isUserAdmin` gate has been removed from agent suggestion rendering so any viewer can see and act on suggestions created by AI agents.

- [**Notifications**]: `maxOrganizations` has been removed from `CrossOrganizationConfig`. Fan-out bounding is now fully server-controlled. Remove this field from any `enableCrossOrganization()` calls â€” the TypeScript type no longer accepts it.

### Breaking Changes

- [**Notifications**]: `CrossOrganizationConfig.maxOrganizations` is removed. Update any `enableCrossOrganization()` calls:

```ts
// Before (v5.0.2-beta.33 / beta.34)
client.enableCrossOrganization({
  organizationIds: ['org-a', 'org-b'],
  maxOrganizations: 10,  // âťŚ no longer accepted
});

// After (v5.0.2-beta.35)
client.enableCrossOrganization({
  organizationIds: ['org-a', 'org-b'],
});
```
