## Overview
- **Key improvements in this series**:
  - 115+ primitive components for building custom comment dialogs
  - Each subcomponent can now be used independently without requiring the full dialog structure
  - Two usage patterns: ID-Based (Standalone) and Context Wrapper
  - Full control over comment dialog layout and styling


## Breaking Changes

- [**Comment Sidebar V2: `MinimalActionsDropdown` Removed**] (beta.38) The `minimal-actions-dropdown-v2` component family was removed from Comment Sidebar V2 — its web component tags, React components, and the `VeltCommentsSidebarV2Wireframe.MinimalActionsDropdown` wireframe slot. Its "mark all read / mark all resolved" actions are now provided by the combined `actions` dropdown, configured declaratively via `minimal-filters`. Apps that did **not** customize this slot need no changes — the default sidebar exposes the equivalent actions through the new dropdown.
  - **Removed (HTML):** `velt-comment-sidebar-minimal-actions-dropdown-v2` (and `-trigger-v2`, `-content-v2`, `-content-mark-all-read-v2`, `-content-mark-all-resolved-v2`) plus their `-wireframe` aliases.
  - **Removed (React):** `VeltCommentSidebarV2MinimalActionsDropdown` (and `Trigger`, `Content`, `ContentMarkAllRead`, `ContentMarkAllResolved`), the `VeltCommentsSidebarV2MinimalActionsDropdown*` wireframe exports, and the `VeltCommentsSidebarV2Wireframe.MinimalActionsDropdown` slot.

  **Before:**
  ```tsx
  <VeltCommentsSidebarV2Wireframe>
    <VeltCommentsSidebarV2Wireframe.MinimalActionsDropdown>
      {/* mark all read / mark all resolved */}
    </VeltCommentsSidebarV2Wireframe.MinimalActionsDropdown>
  </VeltCommentsSidebarV2Wireframe>
  ```

  **After:**
  ```tsx
  // Configure a combined sort + quick "actions" dropdown declaratively:
  <VeltCommentsSidebarV2 minimalFilters={[{ type: 'actions' }]} />

  // Override the dropdown via the FilterDropdown wireframe slot:
  <VeltCommentsSidebarV2Wireframe>
    <VeltCommentsSidebarV2Wireframe.FilterDropdown />
  </VeltCommentsSidebarV2Wireframe>
  ```

- [**Node SDK v1.0.5 — `PartialReactionAnnotation.user` renamed to `from`**] In `@veltdev/node`, the reacting-user field on `PartialReactionAnnotation` was renamed from `user` to `from`. Self-hosting reaction resolvers that call `saveReactions()` must update the field name.

  **Before:**
  ```typescript
  await reactions.saveReactions({
    metadata: { organizationId: 'org-123' },
    reactionAnnotation: {
      'reaction-1': { annotationId: 'reaction-1', icon: 'thumbsup', user: { userId: 'u-1' } }, // old field
    },
  });
  ```

  **After:**
  ```typescript
  await reactions.saveReactions({
    metadata: { organizationId: 'org-123' },
    reactionAnnotation: {
      'reaction-1': { annotationId: 'reaction-1', icon: 'thumbsup', from: { userId: 'u-1' } }, // renamed field
    },
  });
  ```

- [**Python SDK v0.1.12 — `PartialReactionAnnotation.user` renamed to `from_`**] In `velt-py`, the reacting-user attribute on `PartialReactionAnnotation` was renamed from `user` to `from_` (the wire key is `from`). Code that passes `user=PartialUser(...)` must switch to `from_=PartialUser(...)`. Reads stay backward compatible — `from_dict()` accepts either `from` or the legacy `user`, so existing reaction documents resolve with no migration.

  **Before:**
  ```python
  from velt_py.models.reaction import PartialReactionAnnotation
  from velt_py.models.user import PartialUser

  reaction = PartialReactionAnnotation(
      annotation_id="reaction-1",
      icon="thumbsup",
      user=PartialUser(user_id="u-1"),   # serialized as {"user": {...}}
  )
  ```

  **After:**
  ```python
  from velt_py.models.reaction import PartialReactionAnnotation
  from velt_py.models.user import PartialUser

  reaction = PartialReactionAnnotation(
      annotation_id="reaction-1",
      icon="thumbsup",
      from_=PartialUser(user_id="u-1"),  # serialized as {"from": {...}}
  )
  ```

- [**CSS Element Names**] If you applied CSS to comment dialog element selectors, you need to update them to target the new element names.
  - Replace `snippyly-comment-dialog` with `velt-comment-dialog-internal`
  - Replace `app-comment-dialog-*` pattern with `velt-comment-dialog-*-internal`
  - For example: `app-comment-dialog-composer` becomes `velt-comment-dialog-composer-internal`
  - **Important**: Only element names need to be updated. Class names remain unchanged.

## How to Upgrade
### Upgrade Manually
1. Search your CSS files for any element selectors using `snippyly-comment-dialog` or `app-comment-dialog-*` patterns.
2. Update the element selectors using the following replacements:

| Old Element Name | New Element Name |
|------------------|------------------|
| `snippyly-comment-dialog` | `velt-comment-dialog-internal` |
| `app-comment-dialog-composer` | `velt-comment-dialog-composer-internal` |
| `app-comment-dialog-header` | `velt-comment-dialog-header-internal` |
| `app-comment-dialog-body` | `velt-comment-dialog-body-internal` |

3. Deploy the latest version of the Velt SDK to your product.

<Note>
If you are only using class selectors (e.g., `.comment-dialog-composer`), no changes are required.
</Note>

### Upgrade using AI

Use this prompt to update the CSS element selectors in all CSS files for the comment dialog components ONLY.

```
Update CSS element selectors in all CSS files for the comment dialog components ONLY.

Replacements needed:
1. `snippyly-comment-dialog` (element) → `velt-comment-dialog-internal`
2. `app-comment-dialog-{name}` (element) → `velt-comment-dialog-{name}-internal`

SCOPE: Only modify selectors that match these EXACT patterns:
- `snippyly-comment-dialog`
- `app-comment-dialog-*` (e.g., app-comment-dialog-composer, app-comment-dialog-header, etc.)

Do NOT modify any other elements like app-recording-*, app-presence-*, etc.

CRITICAL: Only modify ELEMENT selectors, not class selectors.
- Element selectors have NO dot prefix: `app-comment-dialog-composer { }`
- Class selectors HAVE a dot prefix: `.velt-comment-dialog-composer { }`

Correct transformations:
✅ `snippyly-comment-dialog { }` → `velt-comment-dialog-internal { }`
✅ `app-comment-dialog-composer::part(btn)` → `velt-comment-dialog-composer-internal::part(btn)`
✅ `div > app-comment-dialog-header` → `div > velt-comment-dialog-header-internal`

Do NOT modify:
❌ `.velt-comment-dialog-composer` (class selector - has dot prefix)
❌ `.snippyly-comment-dialog` (class selector - has dot prefix)
❌ `app-recording-player` (different component - out of scope)
❌ `app-presence-avatar` (different component - out of scope)

```
