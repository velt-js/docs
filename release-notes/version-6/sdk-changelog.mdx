### New Features

- [**Core**]: New [`setUnstyledMode()`](/api-reference/sdk/api/api-methods#setunstyledmode) client method turns off Velt's built-in visual styling for headless use, keeping layout and positioning intact by default. Pass `{ keepFunctionalStyles: false }` to strip down to browser defaults, and call with `false` to restore styling. Applies to styles in the page head and shadow roots, including styles injected later.

<Tabs>
<Tab title="React / Next.js">
```tsx
// Remove Velt's visual styling; keep layout/positioning (default)
client.setUnstyledMode(true);

// Strip everything down to raw browser defaults
client.setUnstyledMode(true, { keepFunctionalStyles: false });

// Restore all Velt styling
client.setUnstyledMode(false);
```
</Tab>
<Tab title="Other Frameworks">
```js
Velt.setUnstyledMode(true);
Velt.setUnstyledMode(true, { keepFunctionalStyles: false });
Velt.setUnstyledMode(false);
```
</Tab>
</Tabs>

### Improvements

- [**Self-Hosting**]: New optional `dataRegions` field on the self-hosted config lets you declare which regional data endpoints you provisioned; `setEncryptedData` writes routed to an unprovisioned region now fall back to the base endpoint instead of failing. Adds `asiaNorthEast1` (Tokyo) to the known regions, and leaving `dataRegions` unset keeps current behavior. [Learn more →](/self-host-data/overview)

### Bug Fixes

- [**Comments**]: Comment Sidebar (V1) filters no longer reset unexpectedly — they now survive the Reset button, persist across [`setLocation()`](/api-reference/sdk/api/api-methods#setlocation) changes (only the version filter syncs from location), and are no longer wiped when the host app's `sessionStorage` is full.

- [**Self-Hosting**]: React SDK now stashes `selfHosted.firebaseConfig` before the Velt script loads, so self-hosted React apps reliably apply it on first load.

</Update>
