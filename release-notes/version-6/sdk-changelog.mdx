### Libraries
- `@veltdev/react`

<Update label="6.0.0-beta.4" description="July 02, 2026">

This release brings the Comment Sidebar V2 (`<velt-comments-sidebar-v2>`) to parity with V1 across grouping, filtering, and counting. No new public API methods or breaking changes are included, and the React SDK change is a version bump only.

### Improvements

- [**Comments**]: When grouped by **location**, the Comment Sidebar V2 list now expands the current and additional-location groups while other locations start collapsed; when grouped by **document**, only the current document starts expanded. Status, priority, and custom groupings still start fully expanded, while explicit expand and collapse choices persist across reloads and take precedence in that order over the default; a real location change resets them, while the initial location emitted during a reload preserves them. [Learn more →](/async-collaboration/comments-sidebar/v2/customize-behavior#groupconfig)

- [**Comments**]: The Status filter now starts with **Open** and every **In Progress** status pre-selected as visible, clearable options on first load, matching V1's `setDefaultStatuses`. The seed is skipped when `sessionStorage` state was restored, when a client-provided status filter was supplied, or once you touch the selection; an untouched seed refreshes if the status catalog loads later. [Learn more →](/async-collaboration/comments-sidebar/v2/customize-behavior#default-status-selection)

- [**Comments**]: [`setCommentSidebarFilters()`](/api-reference/sdk/api/api-methods#setcommentsidebarfilters) now maps client filters onto the sidebar's selected-filter state, so they render as checked options and are cleared by **Reset**. Each call replaces selections for keys it includes while preserving omitted keys; facet counts remain absolute within the page-scoped universe, and the Main Filter badge reflects the applied selections. [Learn more →](/async-collaboration/comments-sidebar/v2/customize-behavior#setcommentsidebarfilters)

import { useVeltClient } from '@veltdev/react';

### Bug Fixes

- [**Comments**]: Name-only locations (`setLocation({ locationName: 'Home' })`) now scope sidebar grouping, filtering, page mode, and client filters. Location identity uses `id` when present (including `0`) and otherwise falls back to `locationName`; numeric ids compare consistently with string filter values, and comments without a location appear under **Others**.

- [**Comments**]: People, Involved, Assigned, and Tagged filter options are now keyed strictly by `userId`, so a person referenced by both `userId` and email no longer appears as two options with diverging counts. Options use the person's name with an email fallback; contacts that only have an email do not become filter options.

- [**Comments**]: With `setSidebarButtonCountType('filter')`, the sidebar-button badge now counts the actual filtered list, including an empty list showing `0` and updates after deletion. The same filtered list gates on-page pins when `filterCommentsOnDom` is enabled; the current result is preserved while the sidebar loads and stops gating the badge and pins after the sidebar is removed. [Learn more →](/async-collaboration/comments-sidebar/v2/customize-behavior#sidebarbuttoncounttype)

- [**Comments**]: Selecting **"All"** on the Status field or hitting **Reset** now reveals resolved/terminal comments, matching V1. Clearing a non-status field leaves the resolved flag untouched, and Reset does not re-trigger the Open + In Progress seed.

- [**Comments**]: The default Priority section now shows a **"Not set"** bucket for comments with no priority, matching V1. Fields configured with `includeUnset: false` still omit it.

- [**Comments**]: Client filters pushed via [`setCommentSidebarFilters()`](/api-reference/sdk/api/api-methods#setcommentsidebarfilters) for a field not shown in the Main Filter panel (for example, `version`) now filter the list. Empty selections are ignored, and a field already declared in the panel is not duplicated.

- [**Comments**]: The cross-field `system-filters-operator` (AND/OR) is now applied directly to the sidebar filter engine, including a value set before initialization and changes made at runtime. Within-field matching remains OR.

- [**Comments**]: The Comment Sidebar V2 virtual-scroll list now clips wide rows instead of scrolling horizontally, matching V1's layout.
