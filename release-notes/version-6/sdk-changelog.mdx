<Update label="6.0.0-beta.4" description="July 02, 2026">

This release brings the Comment Sidebar V2 (`<velt-comments-sidebar-v2>`) to parity with V1 across grouping, filtering, and counting. No new public API methods and no breaking changes.

### Improvements

- [**Comments**]: When grouped by **location** or **document**, the Comment Sidebar V2 list now expands only the current page's (or current document's) group by default and collapses the rest, matching V1. User expand and collapse actions are tracked independently and persisted to `sessionStorage`, so a group you open or close stays that way across reloads. [Learn more →](/async-collaboration/comments-sidebar/v2/setup)

- [**Comments**]: The Status filter now starts with **Open** and every **In Progress** status pre-selected as visible, clearable options on first load, matching V1's `setDefaultStatuses`. The seed is skipped when `sessionStorage` state was restored, when a client filter was supplied, or once you touch the selection. [Learn more →](/async-collaboration/comments-sidebar/v2/customize-behavior)

- [**Comments**]: [`setCommentSidebarFilters()`](/api-reference/sdk/api/api-methods#setcommentsidebarfilters) now maps client filters onto the sidebar's own selected-filter state, so they render as checked options, merge per key, and are cleared by **Reset**. Facet option counts stay absolute within the page-scoped universe instead of shrinking when client filters are applied. [Learn more →](/api-reference/sdk/api/api-methods#setcommentsidebarfilters)

### Bug Fixes

- [**Comments**]: Name-only locations (`setLocation({ locationName: 'Home' })`) now scope sidebar grouping, the location filter, and page mode. A single location-identity source of truth keys locations by `id`, falling back to `locationName` for id-less pages, and coerces numeric ids to strings.

- [**Comments**]: People, Involved, Assigned, and Tagged filter options are now keyed strictly by `userId`, so a person referenced by both `userId` and email no longer appears as two options with diverging counts. Options are labeled `name || email`.

- [**Comments**]: With `setSidebarButtonCountType('filter')`, the sidebar-button badge now counts the actual filtered list (including an empty list showing 0) instead of falling back to all annotations. The filter map is preserved during skeleton loading and reset when the sidebar is destroyed.

- [**Comments**]: Selecting **"All"** on the Status field or hitting **Reset** now reveals resolved/terminal comments, matching V1. Clearing a non-status field leaves the resolved flag untouched, and Reset does not re-trigger the Open + In Progress seed.

- [**Comments**]: The default Priority section now shows a **"Not set"** bucket for comments with no priority, matching V1. Fields configured with `includeUnset: false` still omit it.

- [**Comments**]: Client filters pushed via [`setCommentSidebarFilters()`](/api-reference/sdk/api/api-methods#setcommentsidebarfilters) for a field not shown in the Main Filter panel (e.g. `version`) now filter the list by synthesizing a field descriptor for it. Empty selections are not synthesized and declared fields are never duplicated.

- [**Comments**]: The cross-field `system-filters-operator` (AND/OR) is now applied directly to the sidebar filter engine, including a value set before the sidebar initializes. Within-field matching remains OR.

- [**Comments**]: The Comment Sidebar V2 virtual-scroll list now clips wide rows instead of scrolling horizontally, matching V1's layout.

</Update>
