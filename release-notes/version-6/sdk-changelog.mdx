### Libraries
- `@veltdev/react`

- `@veltdev/sdk`

<Update label="6.0.0-beta.1" description="June 23, 2026">

First beta of the modular SDK. The public `Velt` browser API is fully backward compatible — existing browser integrations work unchanged. Approval Engine definition authors should review the breaking `edges[]` contract change below.

### Breaking Changes

- [**Approval Engine**]: Workflow transitions are now expressed through a single unified `edges[]` array. The per-human-node `onReject` shorthand and the top-level `loops[]` array have been removed — a reject path is now an outgoing `on: "reject"` edge, and loop-backs are reject edges marked with `loop`. Already-stored definitions and in-flight executions are unaffected. [Learn more →](/ai/approval-engine/customize-behavior)

### New Features

- [**Core**]: **Modular SDK** — each feature now loads as its own on-demand chunk. Pass `featureAllowList` at init (e.g. `['comment', 'presence']`) to preload only the features you use; omit it to keep the existing behavior of preloading everything. [Learn more →](/api-reference/sdk/models/data-models)

- [**Core**]: New `preload<Feature>()` methods (e.g. `preloadComment()`) to warm a feature chunk ahead of use, or to load tag-only features (`userInvite`, `userRequest`, `videoPlayer`). [Learn more →](/api-reference/sdk/api/api-methods)

- [**Memory**]: `POST /v2/memory/knowledge/search` accepts an optional `includeRules` boolean. When `true`, results merge knowledge chunks and extracted rules into a single relevance-ranked list, each tagged with a `kind` discriminator (`"chunk"` or `"rule"`). Existing requests are unaffected. [Learn more →](/api-reference/rest-apis/v2/memory/knowledge/search)

- [**Approval Engine**]: Definitions now return a read-only `compiled` block (`forwardEdges` and `loops`) so you can render the runtime graph without re-implementing the compiler. [Learn more →](/api-reference/rest-apis/v2/approval-engine/definitions/get-definition)

- [**Approval Engine**]: `waitAll` and `cancelOnQuorum` groups can now be edge sources for collective approve/reject fan-out. [Learn more →](/ai/approval-engine/customize-behavior)

- [**Approval Engine**]: Nodes accept optional `name` and `description` fields for stable labels in visual editors. [Learn more →](/api-reference/rest-apis/v2/approval-engine/definitions/create-definition)

- [**Agents**]: New `rest-api` context-gathering strategy — custom agents can fetch up to 10 of your REST endpoints at execution time and inject the results into the prompt via `{{restApiData}}`.

- [**Agents**]: New `mcp-tools` execution strategy — custom agents can connect to your MCP servers and run a multi-turn tool-calling loop, on both Claude and Gemini.

### Improvements

- [**Core**]: `getXElement()` accessors (e.g. `getCommentElement()`) automatically load their feature chunk and enable the feature — existing code works unchanged. [Learn more →](/api-reference/sdk/api/api-methods)

- [**Core**]: `'crdt'` and `'selection'` are now valid `featureAllowList` keys with their own chunks and preloaders (`preloadCrdt()`, `preloadSelection()`). [Learn more →](/api-reference/sdk/models/data-models)

- [**Agents**]: Partial updates to a custom agent config now deep-merge, so nested fields (e.g. `execution.mcpServers`) are preserved when you update siblings.

### Bug Fixes

- [**Recorder**]: Subtitles and transcription now work even when the recorder chunk wasn't eagerly loaded.

- [**Reactions**]: The current user's reaction pin now reliably shows its active highlight.

- [**Memory**]: Email digest project cards are now deterministic — two digest runs over identical data on the same UTC day produce the same results.

- [**Agents**]: Create and update now return `400` when `instructions` are missing for strategies that require them.

- [**Agents**]: Agent version snapshots no longer leak encrypted auth secrets on the read path.

</Update>
