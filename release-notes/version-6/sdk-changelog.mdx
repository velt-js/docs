### Bug Fixes

- [**Comments**]: With `page-mode="true"` on `<velt-comments-sidebar-v2>`, calling [`setLocation()`](/api-reference/sdk/api/api-methods#setlocation) now keeps every location group in the list (current group expanded on top, others collapsed but still visible) instead of hiding them, and the header Status counts reflect all comments rather than only the current location's. [Learn more →](/async-collaboration/comments-sidebar/v2/customize-behavior#pagemode)

- [**Comments**]: [`setLocation()`](/api-reference/sdk/api/api-methods#setlocation) now always re-expands the current location's group in the Comment Sidebar V2, even if that group was previously collapsed and remembered across a reload, while preserving every other group's remembered collapse state.

- [**Comments**]: The Comment Sidebar V2 navigation button now opens the correct comment in long, virtualized lists. Recycled rows now read their current comment id instead of a cached id from a previous comment.

</Update>
