## Overview
- **Key changes in this series**:
  - Enums in `@veltdev/types` that appear in public method signatures are now `as const` objects with string-literal union types instead of runtime `enum` objects.
  - The published `@veltdev/types` entry chain is now type-only, so type-only symbols can no longer be imported as runtime values.

## Breaking Changes

- [**Types**] 16 enums in `@veltdev/types` that appear in public method signatures were converted from nominal TypeScript `enum` to `as const` object + string-literal union type. You can no longer import and use `EnumName.MEMBER` as a runtime value — pass the equivalent string literal instead, with full type-checking and autocomplete preserved. (Before this release, such an import already resolved to `undefined` at runtime as a phantom value; it is now a compile error that points you to the correct usage.) The affected enums are `CommentStatus`, `ResolverActions`, `CommentResolverSaveEvent`, `AreaStatus`, `DocumentAccessType`, `CommentAccessMode`, `ServerConnectionState`, `ContactListScopeForOrganizationUsers`, `CommentSidebarSystemFiltersOperator`, `RevokeAccessOnType`, `PermissionSource`, `PermissionResourceType`, `UserPermissionAccessRole`, `UserPermissionAccessRoleResult`, `UserIndicatorPosition`, and `UserIndicatorType`.

<Tabs>
<Tab title="React / Next.js">
**Before:**
```tsx
import { PermissionResourceType } from '@veltdev/types';

// PermissionResourceType.DOCUMENT was `undefined` at runtime (phantom value).
const resourceType = PermissionResourceType.DOCUMENT;
```

**After:**
```tsx
import type { PermissionResourceType } from '@veltdev/types';

// Pass the string literal — typed as `PermissionResourceType` ('folder' | 'document' | ...).
const resourceType: PermissionResourceType = 'document';
```

## How to Upgrade

### Upgrade Manually
1. Search your codebase for value imports from `@veltdev/types` of the affected enums (for example, `import { CommentStatus } from '@veltdev/types'`).
2. Change each to a type-only import (`import type { ... } from '@veltdev/types'`) and replace every `EnumName.MEMBER` runtime usage with the equivalent string literal.
3. Deploy the latest version of the Velt SDK to your product.
