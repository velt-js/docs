## Whitelisting Rules for Content Security Policy (CSP)

If you have a Content Security Policy (CSP) enabled in your app configuration, ensure that the following URLs are whitelisted:

### script-src
- `*.velt.dev`
- `*.api.velt.dev`
- `*.firebaseio.com`
- `*.googleapis.com`
- `wss://*.firebaseio.com`
- `wss://*.firebasedatabase.app`

### connect-src
- `*.velt.dev`
- `*.api.velt.dev`
- `*.firebaseio.com` 
- `*.googleapis.com`
- `wss://*.firebaseio.com`
- `wss://*.firebasedatabase.app`

### img-src
- `storage.googleapis.com`
- `firebasestorage.googleapis.com`

### media-src
- `storage.googleapis.com`
- `firebasestorage.googleapis.com`

### Handling `unsafe-eval`
Some Velt features may require `unsafe-eval` which might be restricted by your CSP. If you see an error like `Refused to evaluate a string as JavaScript because 'unsafe-eval' is not an allowed source`, you can use the `enableSafeEval` method as a workaround.

<Tabs>
<Tab title="React / Next.js">
```jsx
client.enableSafeEval();
```
</Tab>
<Tab title="Other Frameworks">
```js
Velt.enableSafeEval();
```
</Tab>
</Tabs>

You can disable it again using `disableSafeEval`.
