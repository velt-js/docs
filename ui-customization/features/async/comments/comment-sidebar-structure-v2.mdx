> **Conventions**
> - The root element is `velt-comments-sidebar-v2` (plural *comments*). All of its sub-primitives use the singular form `velt-comment-sidebar-…-v2`.
> - In React, standalone primitives are exported as `VeltCommentSidebarV2*` (singular). The same parts are also available as wireframe slots under `VeltCommentsSidebarV2Wireframe.*` (plural).
> - **HTML wireframe** elements mirror the React slots 1:1 with a `-wireframe` suffix.
> - **Parent/child** in the wireframe tree is determined **only** by the dot-path extension (`A.B` → `B` is a child of `A`). Visual placement does not imply hierarchy.
> - **(Leaf)** = component has no children in this wireframe.

## Root Props

The Comment Sidebar V2 root element (`<velt-comments-sidebar-v2>` / `VeltCommentsSidebarV2`) accepts a declarative set of props to configure mode, layout, filtering, sorting, grouping, and search.

### Mode / layout

| Property | HTML attribute | Type | Default |
|---|---|---|---|
| `pageMode` | `page-mode` | `boolean` | `false` |
| `focusedThreadMode` | `focused-thread-mode` | `boolean` | `false` |
| `readOnly` | `read-only` | `boolean` | `false` |
| `embedMode` | `embed-mode` | `string \| null` | `null` |
| `floatingMode` | `floating-mode` | `boolean` | `false` |
| `position` | `position` | `'right' \| 'left'` | `'right'` |
| `variant` | `variant` | `string` | `'sidebar'` |
| `forceClose` | `force-close` | `boolean` | `true` |
| `fullScreen` | `full-screen` | `boolean` | `false` |
| `fullExpanded` | `full-expanded` | `boolean` | `false` |
| `shadowDom` | `shadow-dom` | `boolean` | `false` input / **on** effective |

<Note>
**Shadow DOM:** The `shadow-dom` input's literal default is `false`, but the effective rendering default is **on** — the sidebar renders inside a shadow root by default (matching V1) to isolate it from host-page styles. The sidebar only opts out when `shadow-dom="false"` / `disableSidebarShadowDOM()` is set explicitly.
</Note>

## Filter, Sort, Group & Search System

The Comment Sidebar V2 provides a complete, declarative system to filter, sort, group, and search the thread list. Filtering, faceting (per-option counts), sorting, and grouping run deterministically client-side over the loaded annotations. Within a single field, selections combine with **OR**; `filter-operator` controls how different fields combine.

### The three filter surfaces

| Input | Surface | Driven by |
|---|---|---|
| `filters` | **Main Filter** — the full bottom-sheet panel (multi-section, apply/reset). When passed an **object** instead of a `FilterField[]`, it is treated as active selections. | `FilterField[]` or `{ status: […], people: […] }` object |
| `mini-filters` | A **single header funnel dropdown** with one section per field. | `FilterField[]` |
| `minimal-filters` | **Multiple configurable header dropdowns** side-by-side (replaces the single funnel when present). One dropdown per array entry. | `SidebarMinimalFilterConfig[]` |

`FilterField` = `{ field, label?, select?: 'single' | 'multi', valuePath?, … }`. Built-in field IDs: `status`, `priority`, `category`, `people`, `assigned`, `tagged`, `involved`, `location`, `version`, `document`. A custom field is any `field` id plus a `valuePath` dot-path into the annotation.

### The filter-dropdown primitive scoped by `type`

A single primitive — `velt-comment-sidebar-filter-dropdown-v2` / `VeltCommentSidebarV2FilterDropdown` — renders each header dropdown, scoped by its `type`:

| `type` | Driven by | Renders |
|---|---|---|
| `filter` | `fields` (`FilterField[]`) or a single `field` + `label` | category checkbox sections |
| `sort` | `sorts` | sort options (single-select) |
| `quick` | `actions` | quick presets + path predicates (single-select) |
| `actions` | `sorts` **+** `actions` | a combined menu: a sort group and a quick group separated by a divider |
| _(unset)_ | — | default quick presets + configured category sections |

- **`sorts`** entry: `"date"` \| `"unread"` (presets) \| `{ preset, label }` \| `{ label, path, order: 'asc' | 'desc' }` (custom-field sort).
- **`actions`** entry: a preset id (`"open"`, `"resolved"`, `"unread"`, …) \| `{ preset, label }` \| `{ label, path, value }` (single predicate) \| `{ label, conditions: [{ path, value }], operator: 'and' | 'or' }` (multi). `value` is a **literal** (e.g. a userId `"1.1"`). Paths auto-flatten nested arrays (e.g. `comments.taggedUserContacts.contact.userId`).

<Note>
`type: 'filter'` entries' fields must be in the global pool (`filters` / `mini-filters`) to get facet counts. `sort` / `quick` / `actions` entries are self-contained.
</Note>

### Configuration types

```ts
// A single filterable field shown inside a filter panel / dropdown.
interface FilterField {
  field: string;
  label?: string;
  select?: 'single' | 'multi';
  searchable?: boolean;
  showCounts?: boolean;
  icon?: string;
  valuePath?: string;       // dot-path into the annotation for a custom field
  includeUnset?: boolean;
  placeholder?: string;
  groupable?: boolean;
  order?: string[];
  options?: SidebarFilterValue[];
}

// A single header dropdown declared via `minimal-filters`.
interface SidebarMinimalFilterConfig {
  type?: 'filter' | 'sort' | 'quick' | 'actions';
  label?: string;
  field?: string;
  fields?: FilterField[];
  sorts?: (string | SidebarSortConfig)[];
  actions?: (string | SidebarQuickFilterConfig)[];
}

interface SidebarSortConfig {
  label?: string;
  preset?: string;          // 'date' | 'unread' | …
  path?: string;            // custom-field sort
  field?: string;
  order?: 'asc' | 'desc';
}

interface SidebarQuickFilterConfig {
  label?: string;
  preset?: string;          // 'open' | 'resolved' | 'unread' | …
  path?: string;            // single-predicate path
  field?: string;
  value?: any;              // literal value (e.g. a userId "1.1")
  conditions?: SidebarQuickCondition[];   // multi-predicate
  operator?: 'and' | 'or';
}

interface SidebarQuickCondition { path?: string; field?: string; value: any; }

type FilterFieldOperator = 'and' | 'or';
type SidebarFilterDropdownType = 'filter' | 'sort' | 'quick' | 'actions';

// Built-in filterable field IDs.
const BUILT_IN_FILTER_FIELD_IDS = [
  'status', 'priority', 'category', 'people', 'assigned',
  'tagged', 'involved', 'location', 'version', 'document',
] as const;
type BuiltInFilterFieldId = typeof BUILT_IN_FILTER_FIELD_IDS[number];
```

To register a custom filterable field, the SDK exposes these extensibility types:

```ts
interface FacetContext { annotations: CommentAnnotation[]; field: FilterField; }
interface FilterFieldResolver {
  id: string;
  optionSource: 'catalog' | 'scan';
  buildOptions(ctx: FacetContext): SidebarFilterValue[];
  matches(annotation: CommentAnnotation, selectedValueIds: string[]): boolean;
}
```

### Grouping

Grouping turns the filtered + sorted flat list into groups by any field. Enable it via `group-config`:

<Tabs>
<Tab title="React / Next.js">
```jsx
{/* Group by page/location (default when enabled) */}
<VeltCommentsSidebarV2 groupConfig={{ enable: true }} />

{/* Group by a specific field */}
<VeltCommentsSidebarV2 groupConfig={{ enable: true, groupBy: 'status' }} />
```
</Tab>
<Tab title="Other Frameworks">
```html
<!-- Group by page/location (default when enabled) -->
<velt-comments-sidebar-v2 group-config='{"enable":true}'></velt-comments-sidebar-v2>

<!-- Group by a specific field -->
<velt-comments-sidebar-v2 group-config='{"enable":true,"groupBy":"status"}'></velt-comments-sidebar-v2>
```
</Tab>
</Tabs>

The current user's "people" group is relabeled "Me"; the current page/location and an "Others"/"None" catch-all float to deterministic positions.

### Wireframe data variables

The sidebar primitives expose these reactive variables for use inside `<velt-data field="…">` / `velt-if` bindings: `value` (search text / option value), `label`, `count`, `mode` (`'single' | 'multi'` for an option group), `selected`, `group` (with `group.label`, `group.count`), `groupingEnabled`, `groupByOptions`, `chips`, `searchable`, `placeholder`, `isFilterActive`, and `appliedCount`. Bindings update live only when the bound value is a signal — bind dynamic values on the **leaf** wireframe, not its container.
