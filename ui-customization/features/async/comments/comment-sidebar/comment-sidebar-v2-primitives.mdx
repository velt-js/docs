## Overview

The Comment Sidebar V2 Primitives API provides granular components that can be used independently to build completely custom sidebar interfaces. Every section of the V2 sidebar — including the entire filter, sort, group, and search UI — is an independently importable and composable primitive. Each primitive can be used standalone or composed together for maximum customization flexibility.

> **Naming convention:** The root element is `velt-comments-sidebar-v2` (plural *comments*). All of its sub-primitives use the singular form `velt-comment-sidebar-…-v2`. In React, standalone primitives are exported as `VeltCommentSidebarV2*` (singular). The same parts are also available as wireframe slots under `VeltCommentsSidebarV2Wireframe.*` (plural) — see the [Comment Sidebar V2 Wireframes](/ui-customization/features/async/comments/comment-sidebar/comment-sidebar-v2-wireframes) page.

Key features of the V2 architecture:
- **Flat primitive components** — each UI section is individually replaceable
- **Declarative filter / sort / group / search** — configured via root props and decomposed into overridable primitives
- **CDK virtual scroll** — renders only the visible viewport slice for smooth performance on large comment sets
- **Focused-thread view** — opens individual threads in a dedicated view inside the sidebar
- **Conditional styling** — all primitives support `velt-class` conditional styling via `applyConditionalClasses`

For the full root-prop reference and the declarative filter/sort/group/search configuration model, see [Comments Sidebar V2 — Structure](/ui-customization/features/async/comments/comment-sidebar-structure-v2).

## Usage Patterns

### Pattern 1: Drop-in Replacement

Use `VeltCommentsSidebarV2` as a direct replacement for `VeltCommentsSidebar`.

<Tabs>
<Tab title="React / Next.js">
```jsx
<VeltCommentsSidebarV2
  pageMode={false}
  focusedThreadMode={false}
  readOnly={false}
  position="right"
  variant="sidebar"
  forceClose={true}
  onSidebarOpen={(data) => console.log('sidebar opened', data)}
  onSidebarClose={(data) => console.log('sidebar closed', data)}
  onCommentClick={(data) => console.log('comment clicked', data)}
  onCommentNavigationButtonClick={(data) => console.log('nav button clicked', data)}
/>
```
</Tab>
<Tab title="Other Frameworks">
```html
<velt-comments-sidebar-v2
  page-mode="false"
  focused-thread-mode="false"
  read-only="false"
  position="right"
  variant="sidebar"
  force-close="true"
></velt-comments-sidebar-v2>
```
</Tab>
</Tabs>

### Pattern 2: Version Prop on Existing Sidebar

Route the existing `VeltCommentsSidebar` to the V2 implementation by setting `version="2"`.

<Tabs>
<Tab title="React / Next.js">
```jsx
<VeltCommentsSidebar version="2" />
```
</Tab>
<Tab title="Other Frameworks">
```html
<velt-comments-sidebar version="2"></velt-comments-sidebar>
```
</Tab>
</Tabs>

### Pattern 3: Compose from Primitives

Build a fully custom sidebar by composing individual primitives.

<Tabs>
<Tab title="React / Next.js">
```jsx
<VeltCommentsSidebarV2>
  <VeltCommentSidebarV2Skeleton />

    <VeltCommentSidebarV2Header />
    <VeltCommentSidebarV2List />

    <VeltCommentSidebarV2PageModeComposer />
    <VeltCommentSidebarV2FocusedThread />
  </VeltCommentSidebarV2Panel>
</VeltCommentsSidebarV2>
```
</Tab>
<Tab title="Other Frameworks">
```html
<velt-comments-sidebar-v2>
  <velt-comment-sidebar-skeleton-v2></velt-comment-sidebar-skeleton-v2>
  <velt-comment-sidebar-panel-v2>
    <velt-comment-sidebar-header-v2></velt-comment-sidebar-header-v2>
    <velt-comment-sidebar-list-v2></velt-comment-sidebar-list-v2>
```

Root container for the V2 sidebar. Provides shared state context to all child primitives. See [Comments Sidebar V2 — Structure](/ui-customization/features/async/comments/comment-sidebar-structure-v2) for the full root-prop reference (mode/layout, filtering/sorting, dialog/placeholders, virtual scroll, URL navigation, and events).

<Tabs>

```jsx

  pageMode={false}
  focusedThreadMode={false}
  readOnly={false}

  variant="sidebar"

    { type: 'sort', sorts: ['date', 'unread'] },
    { type: 'filter', fields: [{ field: 'status' }, { field: 'assigned' }] },
```
