- *Scope: the **conditional** (stateful) class list is exhaustive across components; the **structural root** classes below are exhaustive for **comments + notifications + core theming** only (other features' structural classes: inspect live with `shadowDom={false}`). </Note>

Every **stateful / conditional** `velt-*` class the SDK toggles: extracted per component, with the **condition** that applies it (the condition *is* the description: "applied when …"). Target these (override with `!important`, R9b) to style state without a wireframe or hook. Prefer the `velt-*` BEM class over the short legacy twin where both appear. Always-on **structural root** classes for in-scope components are listed in their own section near the bottom.

> Conditional classes are exhaustive. Conditions reference internal SDK state expressions; read them as "applied when `expr` is truthy".
>
> **Count: 393 class entries** (one per `` - `velt-…` `` bullet below). If a script reports fewer, it's matching only one pattern: many entries pair a `velt-*` BEM class with its short legacy twin on the same line.

## Additional stateful classes & corrections (in-scope)

Newly verified conditional classes plus corrections to commonly-misremembered names. **Document absences:** three names that are often assumed to be classes are **not** runtime classes.

- `velt-comment-pin-open`: applied to the comment pin when selected (`componentConfig.uiState.commentPinSelected && !annotationDragging`). **There is no `velt-comment-pin-active`**: `velt-comment-pin-open` is the real state class (also listed under **comment-pin** above).
- `velt-reaction-tool-open`: toggled on the reaction tool host when the reaction picker/tooltip is open (`tooltipVisible`); removed when closed.
- `velt-notification-panel--settings-accordion-open`: applied to a settings accordion when expanded (`componentConfig.uiState.settingsAccordionExpanded[id]`). Companion modifiers in the same subtree: `velt-notification-panel--settings-accordion-container-expanded`, `...-content-item-container-selected`.
- `velt-comment-sidebar-minimal-filter-content-item--filter-resolved` / `--filter-unread` / `--sort-unread`: **structural** modifiers applied per dropdown item (one per item, always-on, not condition-toggled). Multi-thread twins: `velt-multi-thread-comment-dialog-minimal-filter-content-item--filter-resolved` / `--filter-unread` / `--sort-unread`.
- **Absence, not CSS classes:** `velt-comment-disabled` and `velt-area-pin-active` do **not** exist as classes; they are the **data attributes** `data-velt-comment-disabled="true"` and `data-velt-area-pin-active="true"`. Style them via the attribute selector (e.g. `[data-velt-area-pin-active="true"]`), not a `.velt-…` class.
