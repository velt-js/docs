<Note>
**Source of truth: exhaustive for what it covers.** Generated from Velt component metadata.

- **If a component isn't on this page, it doesn't exist**: don't import it.
- Never invent or guess a name; use only identifiers that appear verbatim here or that you verified against ground truth and added here.
- *Scope: the complete React primitive (non-wireframe) component surface; each composes over Velt `velt-*` custom elements.*
</Note>

All **491** primitive (non-wireframe) React components the SDK exports: the full set, including the **sub-components** that mirror each wireframe's slot tree. Mount the top-level ones (`VeltComments`, `VeltCommentDialog`, …); compose the sub-components to restructure layouts (see [`Primitives`](/ui-customization/primitives)).

<Tip>
For per-component pages with React and Other Frameworks code examples and screenshots, see the **Components** sections in the sidebar (e.g. [Comment Dialog Primitives](/ui-customization/features/async/comments/comment-dialog/primitives)). You can also explore primitives interactively in our [Storybook](https://storybook.velt.dev/).
</Tip>

> **Two surfaces, both complete:** these **491 React components** are the React client surface. Underneath, the SDK registers **190** public `velt-*` custom-element tags (the vanilla/framework-agnostic surface): the React list is larger because many React components are composition sub-components over the same elements. A sub-component's wireframe twin is the same name + `Wireframe` ([`Wireframe components`](/ui-customization/reference/wireframe-components)). Generated from Velt component metadata and exhaustive.
